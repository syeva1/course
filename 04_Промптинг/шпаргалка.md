# Шпаргалка по промптингу

## Структура промпта

```
Роль → Контекст → Задача → Формат → Пример
```

### Шаблон

```
Ты — [роль].

Контекст: [описание ситуации, аудитории, цели].

Задача: [что конкретно нужно сделать].

Формат: [как должен выглядеть результат].

Пример: [образец желаемого результата].
```

---

## XML-теги

Используй XML-теги, когда промпт сложный и нужна чёткая структура:

```xml
<role>Опытный маркетолог</role>

<context>
Компания АССИСТ+ — HR-агентство по подбору бизнес-ассистентов.
Аудитория — предприниматели с оборотом от 1 млн ₽/мес.
</context>

<task>
Напиши 5 вариантов заголовка для лендинга.
Каждый до 10 слов. Тон — уверенный, но не агрессивный.
</task>

<format>
Нумерованный список. После каждого заголовка — 
1 предложение, почему он работает.
</format>

<example>
1. «Делегируй рутину — займись ростом бизнеса»
   Работает: прямой призыв + конкретная выгода.
</example>
```

### Популярные теги

| Тег | Зачем |
|-----|-------|
| `<role>` | Роль AI |
| `<context>` | Фон и ситуация |
| `<task>` | Что нужно сделать |
| `<format>` | Формат ответа |
| `<example>` | Пример результата |
| `<constraints>` | Ограничения (чего НЕ делать) |
| `<tone>` | Тон и стиль |
| `<audience>` | Целевая аудитория |

## Техники

### Few-shot (покажи пример)
```
Переведи названия продуктов в маркетинговые описания:

Продукт: «Подбор ассистента»
Описание: «Найдём идеального бизнес-ассистента за 5 дней — 
без головной боли с резюме и собеседованиями»

Продукт: «Обучение ассистента»
Описание:
```

### Chain-of-thought (по шагам)
```
Проанализируй эту бизнес-ситуацию. Рассуждай по шагам:
1. Определи ключевую проблему
2. Назови 3 возможные причины
3. Предложи решение для каждой причины
4. Выбери лучшее решение и обоснуй
```

### Итерация
```
→ «Сделай короче, максимум 100 слов»
→ «Добавь цифры и конкретику»
→ «Перепиши для Telegram — неформальный тон, эмодзи»
→ «Вариант 2 лучше всех, развей его»
```

## 5 принципов

1. **Будь конкретным** — не «напиши текст», а «напиши пост на 200 слов для Telegram»
2. **Дай контекст** — кто ты, для кого, зачем
3. **Покажи пример** — один пример лучше десяти объяснений
4. **Укажи формат** — список, таблица, текст, JSON
5. **Итерируй** — доводи ответ до идеала в 2–3 шага
