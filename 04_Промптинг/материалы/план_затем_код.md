# Plan-Act: сначала план, потом код

## Идея

Не проси AI сразу писать код. Сначала пусть составит план, потом — реализует. Это как «семь раз отмерь, один раз отрежь».

## Как это работает

### Шаг 1: ПЛАН (думающая модель)

```
Промпт:
"Проанализируй эту задачу: [описание]

НЕ пиши код.
Создай план реализации:
1. Какие файлы нужны
2. Как данные будут двигаться
3. Крайние случаи (что может пойти не так)
4. Как будем тестировать

Сохрани в plan.md — код пока не пиши."
```

### Шаг 2: КРИТИКА (необязательно, но полезно)

```
Промпт:
"Проверь plan.md на:
- Что мы могли пропустить?
- Есть ли более простой способ?
- Где могут быть проблемы с безопасностью?"
```

### Шаг 3: РЕАЛИЗАЦИЯ

```
Промпт:
"Реализуй Шаг 1 из plan.md.
После каждого шага запусти тесты.
Если тесты падают — остановись и скажи.
Если проходят — переходи к следующему шагу."
```

## Почему это работает

- AI не пытается решить всё сразу (и не путается)
- Ты видишь план ДО написания кода (и можешь поправить)
- Меньше переделок — экономия времени и токенов
- План остаётся как документация

## Стратегия «маленькими шагами»

Вместо одного огромного промпта разбивай на мелкие:

1. Сначала создай компонент
2. Потом настрой стили
3. Потом подключи данные
4. Потом добавь валидацию

**Почему:** каждый маленький шаг легче проверить, и ошибки не накапливаются.

## Когда использовать Plan-Act

| Задача | Нужен план? |
|--------|-------------|
| Поправить одну функцию | Нет |
| Создать новый компонент | Может быть |
| Построить фичу из нескольких файлов | Да! |
| Рефакторинг большого модуля | Обязательно! |
| Новый проект с нуля | 100% да |
