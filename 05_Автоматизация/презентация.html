Какую модель выбрать?

Главное правило: не влюбляйся ни в одну модель

Модели обновляются каждый месяц

Сегодня лучшая — Claude, завтра — GPT, послезавтра — что-то новое. Не привязывайся к названиям. Привязывайся к логике выбора.

4 категории задач

  • Креатив — тексты, посты, письма
  • Код — скрипты, HTML, баги
  • Анализ — отчёты, данные, стратегии
  • Массовые — черновики, переводы

СОВЕТ Начни с Auto — Cursor сам подберёт модель

Как проверять

  • Заходи на lmarena.ai — живой рейтинг моделей
  • Там люди голосуют вслепую за лучший ответ
  • Рейтинг обновляется каждую неделю
  • Не верь блогерам — верь бенчмаркам
АССИСТ+

Метод «Тройной прогон»

Если задача важная — не доверяй одной модели

  • 1 Сделай запрос в первой модели (например Claude)
  • 2 Тот же запрос во второй модели (например GPT)
  • 3 Сравни результаты — возьми лучшее из обоих

Переключить модель: внизу окна чата есть выпадающий список — просто нажми на название текущей модели и выбери другую.

ЗАПОМНИ Модели — это инструменты, как отвёртки. Лучшей нет, есть подходящая.

АССИСТ+

Что такое AI-агент?

Не просто чат — цифровой сотрудник

  • Роль — кто он: ассистент, аналитик, копирайтер
  • Контекст — что знает: файлы, данные, история
  • Задачи — что делает: отвечает, анализирует, пишет
  • Правила — как себя ведёт: тон, границы, формат

Ключевое Агент = Роль + Контекст + Задачи + Правила

АССИСТ+

Агент vs Промпт — разница

Почему агент в 10 раз лучше одноразового запроса

Промпт

  • «Напиши пост про подбор ассистентов»
  • AI выдаёт шаблонный текст без стиля
  • Каждый раз объясняешь: кто ЦА, тон, примеры
  • Результат: «Уважаемые коллеги…» — мусор

Агент-копирайтер

  • «Напиши пост про подбор ассистентов»
  • Агент УЖЕ знает: бренд, аудиторию, тон
  • У него есть примеры лучших постов
  • Результат: текст, который можно публиковать

Аналогия Промпт — фрилансер, которому каждый раз объясняешь бриф. Агент — штатный сотрудник.

Скиллы > Промпты

Тренд 2026 — переиспользуемые компоненты

  • Промпт — текст, который копируют и вставляют. Одноразовый
  • Скилл — папка с инструкциями, скриптами, шаблонами. Переиспользуемый
  • Скиллы можно версионировать через Git
  • Скиллы можно переиспользовать между проектами
  • Скиллы можно улучшать по обратной связи

Формула Агент = базовая конфигурация + набор скиллов

Cursor Rules — пошагово

Создаём файл, который превращает Cursor в сотрудника

  • 1 В проекте создай папку .cursor/rules/
  • 2 Внутри создай файл assistant.mdc
  • 3 Заполни по шаблону ниже
  • 4 Cursor автоматически прочитает и будет следовать
# .cursor/rules/assistant.mdc --- description: "Копирайтер АССИСТ+" globs: ["**/*"] --- ## Роль Ты — копирайтер компании АССИСТ+. Аудитория: предприниматели 25-45 лет. Тон: экспертный, живой, без канцелярита. ## Контекст - Компания подбирает бизнес-ассистентов - Ценности: скорость, качество, забота
АССИСТ+

Скилл в деле — конкретный пример

Скинул задачу → агент знает твой бренд → результат в 10 раз лучше

Без скилла

  • «Напиши лендинг для моего бизнеса»
  • AI: шаблонная страница без УТП
  • Нет твоих цен, отзывов, кейсов
  • Нужно переписывать 80%

Со скиллом «лендинг»

  • «Напиши лендинг»
  • Агент читает: ЦА, УТП, цены, tone of voice
  • Результат: готовая структура + тексты
  • Осталось проверить и запустить

Воркфлоу vs Агенты

Два подхода — когда какой

Воркфлоу — заранее известные шаги

Сначала план → потом текст → потом проверка. Каждый шаг — отдельный запрос. Предсказуемо

Агент — свободный исполнитель

Даёшь задачу + инструменты. AI сам решает, как действовать. Гибко, но менее предсказуемо

ПРАВИЛО Воркфлоу — где можешь. Агент — где задача непредсказуема.

Частые ошибки

Чего избегать

  • Ошибка Слишком общая роль — «будь умным ассистентом»
  • Ошибка Нет примеров — агент не понимает формат ответа
  • Ошибка Нет границ — агент берётся за то, что не умеет
  • Ошибка Нет итераций — создал и забыл, не улучшаешь
  • Ошибка Один мега-агент вместо команды специалистов
  • Ошибка Влюбился в одну модель и не пробуешь другие
