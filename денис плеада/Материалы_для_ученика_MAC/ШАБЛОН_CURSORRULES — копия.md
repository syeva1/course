# 📋 ШАБЛОН: .cursorrules для проекта

> **Назначение:** Этот файл — универсальный шаблон для создания `.cursorrules` в корне проекта.
> 
> **ВАЖНО:** Это деперсонализированный шаблон — любой пользователь может его использовать.
> Заполни разделы о себе и компании, чтобы AI лучше понимал контекст работы.
> 
> Скопируй содержимое из `.cursorrules.example` в файл `.cursorrules` в корне проекта и адаптируй под себя.

---

## 🎯 РЕЖИМЫ РАБОТЫ

**ВАЖНО:** В начале файла установи режим работы:
- `BEGINNER_MODE = true` — режим новичка (безопасный, файлы не удаляются)
- `BEGINNER_MODE = false` — продвинутый режим (полный контроль)

---

## 📝 СОДЕРЖИМОЕ ФАЙЛА .cursorrules

```markdown
# CURSOR RULES для проекта [НАЗВАНИЕ_ПРОЕКТА]

## 🔒 РЕЖИМ РАБОТЫ

# Установи режим новичка (true) или продвинутого (false)
BEGINNER_MODE = true

---

## 🚨 КРИТИЧЕСКИЕ ПРАВИЛА (действуют всегда)

### УДАЛЕНИЕ ФАЙЛОВ И ПАПОК

# В режиме новичка (BEGINNER_MODE = true):
# - ЗАПРЕЩЕНО удалять файлы и папки
# - Вместо удаления: переносить в папку "99_Системное/Архив/" или "00_Inbox/"
# - Если файл точно не нужен — предложить архивирование с датой
# - Всегда спрашивать разрешение перед любыми операциями с файлами

# В продвинутом режиме (BEGINNER_MODE = false):
# - Можно удалять файлы только после явной команды "Удали X"
# - Перед удалением показывать список файлов, которые будут удалены
# - Предлагать альтернативу (архивирование) перед удалением

if BEGINNER_MODE:
    # РЕЖИМ НОВИЧКА: СТРОГИЙ ЗАПРЕТ НА УДАЛЕНИЕ
    - ❌ НИКОГДА не удаляй файлы или папки
    - ✅ Вместо удаления: перенеси в "99_Системное/Архив/[ДАТА]/"
    - ✅ Если файл не подходит никуда — перемести в "00_Inbox/"
    - ✅ Всегда объясняй, куда и почему перемещаешь файл
    - ✅ Создавай папку архива автоматически, если её нет
else:
    # ПРОДВИНУТЫЙ РЕЖИМ: УДАЛЕНИЕ С ПОДТВЕРЖДЕНИЕМ
    - ⚠️ Удаляй файлы ТОЛЬКО после явной команды "Удали X"
    - ⚠️ Перед удалением покажи список файлов
    - ✅ Предлагай альтернативу (архивирование) перед удалением
    - ✅ Всегда спрашивай подтверждение перед удалением

## 📂 СТРУКТУРА ПРОЕКТА

# Методология "Система Папок" (Context-First подход)
# Используй логику из @ШАБЛОН_БИЗНЕС_СТРУКТУРЫ.md

КОРНЕВАЯ_ПАПКА = "[НАЗВАНИЕ_ПРОЕКТА]"

СТРУКТУРА_ПАПОК (по методологии "Система Папок"):
  - "00_Inbox/" — входящий хаос, файлы перед сортировкой (ВСЕГДА ПЕРВАЯ)
  - "01_[ОСНОВНАЯ_ПАПКА_1]/" — опиши назначение (адаптируй под бизнес)
  - "02_[ОСНОВНАЯ_ПАПКА_2]/" — опиши назначение
  - "03_[ОСНОВНАЯ_ПАПКА_3]/" — опиши назначение
  - "04_[ОСНОВНАЯ_ПАПКА_4]/" — опиши назначение (если нужно)
  - "05_[ОСНОВНАЯ_ПАПКА_5]/" — опиши назначение (если нужно)
  - "99_Системное/" — технические файлы, промпты, настройки (ВСЕГДА ПОСЛЕДНЯЯ)
    - "99_Системное/Архив/" — архив удаленных/старых файлов (формат: YYYY-MM-DD/)
    - "99_Системное/Промпты/" — промпты и инструкции для AI
    - "99_Системное/Настройки/" — конфигурационные файлы

ПРИНЦИПЫ "СИСТЕМЫ ПАПОК":
  1. Единая точка входа — всё начинается с одной корневой папки
  2. Inbox (00_Inbox) — всегда есть папка для входящего хаоса
  3. Нумерация — папки пронумерованы (01_, 02_...) для фиксации порядка
  4. README — в каждой папке есть инструкция для AI (README.md)
  5. Системное (99_) — технические файлы лежат отдельно, всегда последние

ПРАВИЛА_РАБОТЫ_С_ПАПКАМИ:
  - Всегда создавай README.md в новых папках (с разделами: 🎯 О папке, 📂 Содержимое, ⚙️ Как использовать)
  - Используй нумерацию для фиксации порядка (01_, 02_...)
  - Папка "99_Системное" всегда последняя (99_)
  - Папка "00_Inbox" всегда первая (00_)
  - Новые файлы сначала попадают в "00_Inbox/", потом распределяются
  - Внутри каждой папки могут быть подпапки (тоже с README.md)

## 📋 ПРАВИЛА РАБОТЫ С ФАЙЛАМИ

### СОЗДАНИЕ ФАЙЛОВ
- Всегда создавай файлы в правильных папках согласно структуре "Системы Папок"
- Новые файлы сначала в "00_Inbox/", потом распределяй по структуре
- Используй понятные названия файлов (без спецсимволов, кроме _ и -)
- Для дат используй формат: YYYY-MM-DD
- При создании новой папки сразу создавай README.md с описанием (🎯 О папке, 📂 Содержимое, ⚙️ Как использовать)

### РЕДАКТИРОВАНИЕ ФАЙЛОВ
- Всегда читай файл перед редактированием
- Не используй плейсхолдеры ("// ...", "остальное без изменений")
- Сохраняй полное содержимое файлов при операциях с ними
- Не удаляй содержимое файлов без явной команды

### ПЕРЕМЕЩЕНИЕ ФАЙЛОВ
- Перед перемещением объясняй, куда и почему
- Сохраняй полное содержимое файла при перемещении
- Обновляй ссылки в других файлах, если они есть
- Создавай README.md в новой папке, если её нет

### АРХИВИРОВАНИЕ ФАЙЛОВ
- Формат папки архива: "99_Системное/Архив/YYYY-MM-DD/" (например: 2025-01-25/)
- Сохраняй структуру папок при архивировании
- Добавляй дату архивирования в название папки (формат: YYYY-MM-DD)
- Создавай README.md в папке архива с описанием содержимого
- Если папка архива не существует — создай её автоматически
- В режиме новичка ВСЕГДА используй архивирование вместо удаления
```

   - **Можно заполнять постепенно** — начинай с основного, остальное добавляй позже
7. **Адаптируй стиль общения:**
   - Используй информацию из раздела "Психологический профиль" для настройки стиля
   - Учитывай контекст компании при принятии решений
8. **Сохрани файл** и перезагрузи Cursor

**ВАЖНО:** Структура папок должна соответствовать методологии "Система Папок":
- `00_Inbox/` — всегда первая (входящий хаос)
- `01_`, `02_`, `03_`... — нумерованные основные папки (5-7 штук)
- `99_Системное/` — всегда последняя (технические файлы, архив)
