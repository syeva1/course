# Инструкция: Запуск бота локально

> **Модуль 6:** API Интеграции  
> **Время:** 5-10 минут

---

## Перед запуском

Убедись что у тебя есть:
- [ ] Python 3.9 или выше
- [ ] Токен Telegram бота (из BotFather)
- [ ] API ключ AssemblyAI
- [ ] Все файлы скопированы в проект

---

## Пошаговая инструкция

### Шаг 1: Проверить Python

Открой терминал в Cursor (View → Terminal) и выполни:

```bash
python --version
```

Должно вывести версию Python 3.9 или выше:
```
Python 3.11.4
```

**Если Python не найден** — смотри раздел "Решение проблем".

---

### Шаг 2: Установить зависимости

В терминале выполни:

```bash
pip install -r requirements.txt
```

Ты увидишь установку пакетов:
```
Collecting python-telegram-bot==21.7
Collecting aiohttp>=3.9.0
Successfully installed ...
```

---

### Шаг 3: Настроить config.py

1. Переименуй `config_example.py` → `config.py`
2. Открой `config.py`
3. Заполни свои токены:

```python
TELEGRAM_BOT_TOKEN = "твой_токен_от_botfather"
ASSEMBLYAI_KEY = "твой_ключ_от_assemblyai"
ADMIN_ID = твой_telegram_id
```

4. Сохрани файл

---

### Шаг 4: Создать папку Распределение

В терминале:

```bash
mkdir Распределение
```

Или создай папку вручную в Cursor.

---

### Шаг 5: Запустить бота

В терминале:

```bash
python bot.py
```

Если всё настроено правильно, увидишь:
```
🚀 Запуск бота...
📁 Папка Распределение готова: ./Распределение
✅ Бот запущен! Нажми Ctrl+C для остановки.
```

---

### Шаг 6: Проверить в Telegram

1. Открой Telegram
2. Найди своего бота (по username)
3. Напиши `/start`
4. Бот должен ответить приветствием

---

## Решение проблем

### Python не найден

**Ошибка:**
```
'python' is not recognized as an internal or external command
```

**Решение для Windows:**
1. Скачай Python с [python.org](https://www.python.org/downloads/)
2. При установке отметь **"Add Python to PATH"**
3. Перезапусти терминал

**Решение для Mac:**
```bash
brew install python
```

Или скачай с [python.org](https://www.python.org/downloads/)

---

### pip не найден

**Ошибка:**
```
'pip' is not recognized
```

**Решение:**
```bash
python -m pip install -r requirements.txt
```

---

### Модуль не найден (ModuleNotFoundError)

**Ошибка:**
```
ModuleNotFoundError: No module named 'telegram'
```

**Решение:**
```bash
pip install python-telegram-bot==21.7
```

Или запусти установку заново:
```bash
pip install -r requirements.txt
```

---

### Токен неверный (Unauthorized)

**Ошибка:**
```
telegram.error.Unauthorized: The token is invalid
```

**Решение:**
1. Открой BotFather
2. Напиши `/token`
3. Выбери своего бота
4. Скопируй токен заново
5. Вставь в config.py (в кавычках!)
6. Перезапусти бота

---

### config.py не найден

**Ошибка:**
```
ModuleNotFoundError: No module named 'config'
```

**Решение:**
1. Убедись что файл называется `config.py` (не `config_example.py`)
2. Убедись что файл в той же папке что и `bot.py`

---

### Бот не отвечает

**Проблема:** Написал боту, но он молчит

**Решение:**
1. Проверь что терминал показывает "Бот запущен"
2. Проверь что нет ошибок в терминале
3. Проверь что пишешь правильному боту (по username)
4. Попробуй написать `/start`

---

### Папка Распределение не создаётся

**Решение:**
Создай вручную или убедись что путь в config.py правильный:
```python
DISTRIBUTION_FOLDER = "./Распределение"
```

---

## Как остановить бота

В терминале нажми:
```
Ctrl + C
```

Увидишь:
```
^C
```

Бот остановлен.

---

## Как запустить снова

```bash
python bot.py
```

Бот получит все сообщения, которые пришли пока он был выключен (если прошло меньше 24 часов).

---

## Советы

1. **Держи терминал открытым** — бот работает пока терминал открыт
2. **Не закрывай ноутбук** — бот остановится
3. **Проверяй логи** — файл `bot.log` в папке Распределение
4. **Перезапускай при ошибках** — Ctrl+C, потом `python bot.py`
