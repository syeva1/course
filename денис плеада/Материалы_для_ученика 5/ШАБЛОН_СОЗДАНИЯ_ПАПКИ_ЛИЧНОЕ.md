## 🚀 КАК ИСПОЛЬЗОВАТЬ (для ученика)

### Шаг 1: Открой режим "ПЛАН"

1. В Cursor нажми `Cmd+I` (Composer)
2. В правом верхнем углу выбери режим **"ПЛАН"** (Plan)
3. Перетащи этот файл в окно чата (или используй `@` и найди файл)

### Шаг 2: AI начнет задавать вопросы

AI прочитает инструкцию и начнет задавать вопросы:
- О тебе (имя, возраст, профессия)
- О бизнесе (если есть)
- О потребностях (какие сферы жизни важны)

**Важно:** 
- ✅ **Отвечай честно и подробно** — от этого зависит качество структуры!
- ✅ **Можешь выбирать несколько вариантов** — если AI спрашивает "какие сферы жизни важны", можешь перечислить несколько
- ✅ **Можешь комбинировать варианты** — например, "детальная структура для здоровья, простая для хобби"
- ✅ **Можешь добавлять свои варианты** — не ограничивайся предложенными вариантами

### Шаг 3: Утверди план

AI покажет тебе план структуры папок:
- Какие папки будут созданы
- Почему именно эти папки
- Как они организованы

**Твоя задача:** Проверь план и скажи, что изменить (если нужно).

### Шаг 4: Получи результат

После утверждения AI создаст:
- ✅ Все папки согласно плану
- ✅ README для каждой папки
- ✅ Главный README в корне
- ✅ Главный файл о себе

**Готово!** Теперь у тебя есть структурированная папка "Личное".

## 💡 ЧТО ТЫ УЗНАЕШЬ ОБ ИНТЕРФЕЙСЕ

**Режим "ПЛАН" vs обычный чат:**

- **Обычный чат (`Cmd+L`):** AI сразу выполняет задачу
- **Режим "ПЛАН" (`Cmd+I` → Plan):** AI сначала создает план, задает вопросы, потом выполняет

**Когда использовать "ПЛАН":**
- Сложные задачи с множеством шагов
- Когда нужно учесть индивидуальные особенности
- Когда важна структура и организация

**Когда использовать обычный чат:**
- Простые вопросы
- Быстрые задачи
- Когда знаешь точно, что нужно

## ⚠️ ВАЖНЫЕ ПРАВИЛА

### ✅ ОБЯЗАТЕЛЬНО:

1. **Задавай вопросы последовательно** — не все сразу
2. **Явно указывай, что можно выбрать несколько вариантов** — не ограничивай ответы одним вариантом
3. **Поощряй развернутые ответы** — "Можешь рассказать подробнее" или "Можешь выбрать несколько вариантов"
4. **Адаптируй структуру под ответы** — не используй шаблонные папки, если они не релевантны
5. **Создавай README для каждой папки** — это важно для навигации
6. **Используй только информацию из ответов** — не придумывай факты
7. **Покажи план перед выполнением** — дай возможность скорректировать

### ❌ НЕ ДЕЛАЙ:

1. **НЕ создавай папки "на всякий случай"** — только то, что реально нужно
2. **НЕ придумывай информацию** — используй только то, что сказал человек
3. **НЕ создавай слишком глубокую структуру** — если человек не просил
4. **НЕ пропускай этап планирования** — всегда показывай план перед выполнением
