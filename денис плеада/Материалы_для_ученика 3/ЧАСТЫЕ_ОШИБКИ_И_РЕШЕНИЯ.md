## ОШИБКИ СТРУКТУРЫ

### Ошибка 1: Нет папки агента

**Проблема:** Создали только файл `.mdc`, но не создали папку для контекста.

**Пример:**
```
.cursor/rules/копирайтер.mdc    ← Есть
агенты/КОПИРАЙТЕР/              ← Нет!
```

**Почему плохо:** Агент не может сохранять примеры и логи. Нет места для самообучения.

**Решение:**
```
агенты/КОПИРАЙТЕР/
├── _КОНТЕКСТ/
│   └── примеры/
├── _ЛОГИ/
└── README.md
```

---

### Ошибка 2: Неправильное имя файла .mdc

**Проблема:** Имя файла не совпадает с командой вызова.

**Пример:**
```
Файл: .cursor/rules/marketing_agent.mdc
Команда в файле: @маркетолог
```

**Почему плохо:** Cursor не найдёт агента при вызове `@маркетолог`.

**Решение:**
```
Файл: .cursor/rules/маркетолог.mdc
Команда: @маркетолог
```
Имя файла = имя команды (без @).

---

### Ошибка 3: Нет README.md в папке агента

**Проблема:** Папка агента без описания.

**Пример:**
```
агенты/АНАЛИТИК/
├── _КОНТЕКСТ/
└── _ЛОГИ/
```

**Почему плохо:** Непонятно, что это за агент и как его использовать.

**Решение:**
```markdown
# АНАЛИТИК

## Назначение
Анализ данных и создание отчётов.

## Команда
`@аналитик`

## Задачи
1. Сбор данных
2. Анализ метрик
3. Создание отчётов

## Как использовать
`@аналитик создай отчёт за неделю`
```

### Ошибка 4: Слишком длинные инструкции

**Проблема:** Инструкции на 10+ страниц, агент путается.

**Пример:**
```markdown
## ИНСТРУКЦИИ

При создании поста:
1. Сначала изучи целевую аудиторию...
   1.1. Определи возраст...
   1.2. Определи пол...
   1.3. Определи интересы...
   [... ещё 50 пунктов ...]
```

**Почему плохо:** AI теряет фокус. Слишком много деталей = хаос.

**Решение:**
```markdown
## ИНСТРУКЦИИ

При создании поста:
1. Изучить бриф
2. Написать черновик
3. Проверить по чеклисту
4. Отправить на проверку

### Чеклист поста
- [ ] Есть хук (первые 5 слов)
- [ ] Есть CTA
- [ ] Длина 100-150 слов
```

Держи инструкции короткими. Детали — в отдельных файлах.

### Ошибка 5: Нет секции "НЕ входит"

**Проблема:** Агент берётся за всё подряд.

**Пример:**
```markdown
## ЗАДАЧИ
1. Писать посты
2. Анализировать метрики
3. Общаться с клиентами
4. Настраивать рекламу
5. Делать дизайн
6. Вести переговоры
```

**Почему плохо:** Агент делает всё плохо, вместо того чтобы делать что-то хорошо.

**Решение:**
```markdown
## ЗАДАЧИ

### Основные:
1. Писать посты для Telegram
2. Адаптировать под разные площадки

### НЕ входит:
- ❌ Анализ метрик (это → @аналитик)
- ❌ Дизайн (это → @дизайнер)
- ❌ Настройка рекламы (это → @маркетолог)
```

### Ошибка 6: Нет примеров использования

**Проблема:** Агент не понимает, что от него хотят.

**Пример:**
```markdown
## ЗАДАЧИ
Писать продающие посты.

## ИНСТРУКЦИИ
Пиши хорошо.
```

**Почему плохо:** "Хорошо" — понятие растяжимое. Нет образца.

**Решение:**
```markdown
## ПРИМЕРЫ

### Пример 1: Продающий пост

**Запрос:** 
"Напиши пост о запуске курса по AI"

**Результат:**
---
🚀 Через 7 дней ты будешь думать по-другому.

Не про AI. Про то, как ты работаешь.

Курс "CURSOR: Экзоскелет для ума" — это:
• 5 модулей практики
• 0 воды
• Результат с первого урока

→ Записаться: [ссылка]
---
```

### Ошибка 7: Нет системы самообучения

**Проблема:** Агент не улучшается со временем.

**Пример:**
```markdown
## ЗАДАЧИ
Писать посты.

## ИНСТРУКЦИИ
Получил задачу → написал пост → готово.
```

**Почему плохо:** Агент повторяет одни и те же ошибки.

**Решение:**
```markdown
## САМООБУЧЕНИЕ

### После каждой задачи:
1. Запросить оценку:
   - ✅ Отлично
   - ⚠️ Нужны правки  
   - ❌ Не то

2. При "Отлично" → сохранить в `_КОНТЕКСТ/примеры/`

3. Логировать в `_ЛОГИ/[дата].md`

### Правило:
Если запрос имеет логику → сохранить в контекст.
```

### Ошибка 8: Агент не спрашивает, когда непонятно

**Проблема:** Агент додумывает за пользователя.

**Пример:**
```
Пользователь: "Напиши пост"
Агент: [пишет пост о чём-то рандомном]
```

**Почему плохо:** Результат не соответствует ожиданиям.

**Решение:**
```markdown
## УТОЧНЯЮЩИЕ ВОПРОСЫ

При недостаточном контексте:
1. Определить, чего не хватает
2. Задать 2-3 вопроса (не больше!)
3. Дождаться ответа

### Когда спрашивать:
- Не указана тема
- Не указан формат
- Противоречивые требования

### Пример:
"Уточню перед написанием:
1. Какая тема поста?
2. Какой формат? (короткий/длинный)"
```

### Ошибка 9: Нет правил безопасности

**Проблема:** Агент может удалить важные файлы или сделать необратимые действия.

**Пример:**
```markdown
## ЗАДАЧИ
Организовать файлы и удалить дубликаты.
```

**Почему плохо:** Агент удалит файлы без подтверждения.

**Решение:**
```markdown
## ПРАВИЛА РАБОТЫ

### Всегда:
- ✅ Показать план перед действиями
- ✅ Запросить подтверждение на удаление
- ✅ Логировать все изменения

### Никогда:
- ❌ НЕ удалять файлы без подтверждения
- ❌ НЕ делать необратимые действия
- ❌ НЕ менять структуру без плана
```

## ОШИБКИ ИСПОЛЬЗОВАНИЯ

### Ошибка 10: Слишком абстрактный запрос

**Проблема:** Пользователь даёт размытое ТЗ.

**Пример:**
```
"@копирайтер напиши что-нибудь интересное"
```

**Почему плохо:** Агент не знает, что именно нужно.

**Решение:**
```
"@копирайтер напиши пост о запуске курса:
- Тема: CURSOR: Экзоскелет для ума
- Формат: 100-150 слов
- Стиль: дружелюбный, с юмором
- CTA: записаться на курс"
```

---

### Ошибка 11: Нет обратной связи

**Проблема:** Пользователь не оценивает результат.

**Пример:**
```
Агент: "Вот пост: [текст]. Оцените?"
Пользователь: "Ок" [уходит]
```

**Почему плохо:** Агент не знает, хорошо ли он сделал. Не может улучшиться.

**Решение:**
```
Агент: "Вот пост: [текст]. Оцените?"
Пользователь: "✅ Отлично! Особенно понравился хук."
```

или

```
Пользователь: "⚠️ Нужны правки: слишком длинный, сократи"
```

### Ошибка 12: Игнорирование логов

**Проблема:** Логи копятся, но никто их не читает.

**Пример:**
```
_ЛОГИ/
├── 2024-01-15.md
├── 2024-01-16.md
├── 2024-01-17.md
[... 50 файлов, никто не читает ...]
```

**Почему плохо:** Теряются уроки и паттерны ошибок.

**Решение:**
Раз в неделю:
1. Просмотреть логи
2. Найти повторяющиеся проблемы
3. Обновить правила агента
4. Добавить успешные кейсы в `_КОНТЕКСТ/примеры/`

### Ошибка 13: Один агент на всё

**Проблема:** Создали одного "универсального" агента.

**Пример:**
```markdown
# УНИВЕРСАЛЬНЫЙ ПОМОЩНИК

## ЗАДАЧИ
- Писать посты
- Анализировать данные
- Общаться с клиентами
- Вести бухгалтерию
- Делать дизайн
- Управлять проектами
```

**Почему плохо:** "Мастер на все руки — мастер никуда". Агент делает всё плохо.

**Решение:**
Создай несколько специализированных агентов:
- `@копирайтер` — только тексты
- `@аналитик` — только данные
- `@дизайнер` — только визуал

### Ошибка 14: Нет взаимодействия между агентами

**Проблема:** Агенты работают изолированно.

**Пример:**
```
@копирайтер написал пост
[пост остался в чате, никуда не передан]
@дизайнер не знает, что нужен визуал
```

**Почему плохо:** Потеря информации, дублирование работы.

**Решение:**
```markdown
## СВЯЗЬ С КОМАНДОЙ

### Передача результатов:
- После написания поста → передать @дизайнер для визуала
- После создания визуала → передать @маркетолог для публикации

### Формат передачи:
"@дизайнер нужен визуал для поста: [ссылка на пост]"
```

### Ошибка 15: Забыли про контекст

**Проблема:** Агент не использует накопленные примеры.

**Пример:**
```
_КОНТЕКСТ/примеры/
├── пост_1.md     # Отличный пост
├── пост_2.md     # Отличный пост
[... агент их игнорирует и пишет с нуля ...]
```

**Почему плохо:** Агент не учится на своих успехах.

**Решение:**
```markdown
## ИНСТРУКЦИИ

При создании поста:
1. **Сначала** изучить примеры в `_КОНТЕКСТ/примеры/`
2. Использовать успешные паттерны
3. Написать пост
4. Сравнить с примерами
```
