# Шпаргалка — Вайб-кодинг

## Процесс

```
Идея → README.md → Plan Mode → Agent Mode → Тест → Коммит → Повторить → Деплой
```

## Шаблон запроса для лендинга

```
Создай лендинг для [тип бизнеса].

Секции:
1. Герой-блок: заголовок "[текст]", подзаголовок, кнопка CTA
2. Услуги: 3 карточки с иконками
3. О нас: текст + фото
4. Отзывы: 3 карточки
5. Контакты: форма + телефон + соцсети

Дизайн: [цвета], [шрифт], адаптивный
```

## Шаблон запроса для калькулятора

```
Сделай калькулятор стоимости [чего].

Параметры:
- [Параметр 1]: выпадающий список / ползунок / чекбокс
- [Параметр 2]: ...
- [Параметр 3]: ...

Формула: [описание]
Кнопка результата → ссылка на Telegram
Дизайн: [стиль]
```

## Стратегия маленьких шагов

```
❌ «Сделай весь сайт целиком»

✅ «Добавь форму»
✅ «Сделай адаптив»
```

## Git-команды

```bash
git init                    # Создать репо
git add .                   # Добавить все файлы
git commit -m "Описание"    # Зафиксировать
git push                    # Отправить на GitHub
git log --oneline           # История коммитов
git checkout HEAD~1 -- .    # Откатить к предыдущему
```

## Деплой на GitHub Pages

```bash
# 1. Создай репо на github.com
# 2. Пуш код
git remote add origin https://github.com/USER/REPO.git
git push -u origin main

# 3. Settings → Pages → Source: main → / (root) → Save
# 4. Через 1-2 минуты: https://USER.github.io/REPO
```

## Деплой на Vercel

```bash
# 1. vercel.com → Sign up с GitHub
# 2. Import → выбери репо
# 3. Deploy → готово
# URL: project-name.vercel.app
```

## Полезные промпты для отладки

```
«Белый экран. Ошибка в консоли: [текст ошибки]»
«Кнопка [название] не работает, проверь обработчик»
«На мобилке текст вылезает за экран, исправь адаптив»
«Сделай скриншот [описание проблемы]» + перетащи изображение
```
