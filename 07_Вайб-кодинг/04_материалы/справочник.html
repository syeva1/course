Ключевые правила

Топ-правила для быстрого запуска

  • Один чат = одна задача. Не смешивай темы в одном разговоре
  • Начинай с MVP. Минимум функций, максимум скорости
  • Описывай ЧТО, а не КАК. AI сам выберет реализацию
  • Фиксируй версии. Git commit после каждого рабочего состояния
  • Читай дифф. Всегда проверяй, что AI изменил
  • Главное правило Сложную задачу разбивай на простые шаги

Идея в план и большие проекты

Декомпозиция — ключ к успеху

  • Промпт: «Разбей эту идею на 5-7 этапов. Каждый этап — один чат»
  • PRD — Product Requirements Document. Создай его до начала кода
  • AI забывает всё при новом чате — поэтому нужны файлы: docs/, plan.md
  • Итерации: сделай базу — покажи — собери фидбек — улучши
  • Совет Для проектов больше 1 дня — обязательно используй Git

Сравнение инструментов

Cursor vs альтернативы 2026

  • Cursor — лучший для кода и проектов. Pro $20/мес
  • Windsurf — похож на Cursor, бесплатный тариф щедрее
  • Bolt / Lovable — визуальные конструкторы, проще для новичков
  • Claude Code — терминальный агент, для продвинутых
  • Replit — онлайн-IDE с AI, для быстрых прототипов
  • Вывод Для серьёзной работы — Cursor. Для экспериментов — Bolt/Lovable

Деплой

Публикация проектов в интернет

  • GitHub Pages — бесплатный хостинг для статических сайтов
  • 1Создай репозиторий на GitHub
  • 2Включи Pages в Settings > Pages > Source: main
  • 3Сайт доступен по адресу username.github.io/repo
  • VPS — для ботов, бэкендов и сложных проектов
  • Совет Для статики — GitHub Pages. Для всего остального — VPS

FAQ

Ответы на частые вопросы

  • С чего начать первый проект? — Лендинг или простой калькулятор. Задеплой на GitHub Pages
  • Как выбрать между Cursor и Bolt? — Cursor для проектов, Bolt для быстрых прототипов
  • VPS — это дорого? — От $5/мес. Для обучения можно использовать бесплатные Railway/Render
  • Git обязателен? — Для деплоя на GitHub Pages — да. В целом, очень рекомендуем
  • Можно ли заработать на вайб-кодинге? — Да. Лендинги, боты, автоматизации — востребованы
  • Что делать, если проект сломался? — git log + git checkout для отката к рабочей версии
