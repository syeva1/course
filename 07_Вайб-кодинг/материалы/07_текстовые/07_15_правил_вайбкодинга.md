# 15 правил для быстрого запуска проектов

> Проверенные правила из практики вайб-кодинга.
> Источник: @misha_davai_po_novoi

---

## 1. Не начинайте с нуля
Используйте шаблоны с GitHub. Зачем писать с нуля, если есть готовое? Скажи Cursor: «Найди шаблон для [тип проекта] на GitHub».

## 2. Доверьте рутину AI-агенту
Cursor Agent Mode — пусть AI создаёт файлы, пишет код, исправляет баги. Ты — режиссёр, он — исполнитель.

## 3. Ищите идеи через AI
Не знаешь, что создать? Спроси: «Какие проблемы малого бизнеса можно решить простым веб-приложением?»

## 4. Одна задача — один чат
Не мешай разные задачи в одном чате Cursor. Начал новую задачу — новый чат (Cmd+L).

## 5. Тестируйте чаще, исправляйте быстрее
После каждого шага — открой в браузере, проверь. Нашёл баг → сразу в Cursor → исправил → дальше.

## 6. Не стремитесь к идеалу с первой попытки
MVP (минимальный продукт) > идеальный продукт. Сначала пусть работает — потом красиво.

## 7. Говорите, а не печатайте
Голосовой ввод (Wispr Flow или встроенный) быстрее клавиатуры. Описывайте задачу словами.

## 8. Вдохновляйтесь чужим кодом на GitHub
Нашёл классный проект → клонируй → покажи Cursor → «Сделай как здесь, но для моего бизнеса».

## 9. Копируйте ошибки в чат с AI
Ошибка в консоли → выделить → скопировать → вставить в Cursor. Не описывай — покажи.

## 10. Сохраняйте историю чатов
Удачный промпт = ценный актив. Сохраняй в отдельный файл или используй как .mdc правило.

## 11. Держите ключи и пароли в .env
**Никогда** не пиши токены и пароли в коде. Только `.env`. Добавь в `.gitignore` до первого коммита.

## 12. Регулярно делайте git commit
После каждого рабочего шага:
```bash
git add . && git commit -m "что сделал"
```
Сломал — откатился. Без коммитов — потерял всё.

## 13. Разворачивайте проект как можно раньше
Не жди идеала. Задеплой на Vercel/GitHub Pages на 2-й день. Показывай людям, собирай фидбэк.

## 14. Коллекционируйте удачные промпты
Создай папку `prompts/` или файл `prompts.md`. Записывай всё, что сработало. Это твоя база знаний.

## 15. Получайте удовольствие от процесса
Вайб-кодинг — это творчество. Не код ради кода, а решение проблем. Если не весело — ты делаешь что-то не то.
