# Как работать с большими проектами в Cursor

> AI забывает всё при новом чате. Вот как с этим жить.
> Источник: вебинар Андрея Погорелого (Нейроцех)

---

## Проблема

AI — это программист, который **каждый день приходит новый**. Вчерашний уволился, сегодняшний ничего не знает. Если ты не записал, что было — придётся объяснять заново.

## Решение: записывай всё

### 1. Перед началом — план

```
Составь план работы: что будешь делать, в каком порядке,
какие файлы затронешь. Сохрани план в файл plan.md
```

### 2. После каждого шага — итоги

```
Запиши, что конкретно сделал, что изменилось, какие решения
принял. Сохрани в progress.md
```

### 3. В новой сессии — дай контекст

```
Прочитай plan.md и progress.md. Вот что мы делали раньше,
вот где остановились. Продолжай со следующего шага.
```

AI прочитает документацию и подхватит проект.

---

## Декомпозиция: дроби на части

**❌ Плохо:** «Создай систему анализа тендеров с парсингом, AI-обработкой, дашбордом и отчётами»

**✅ Хорошо:**
1. «Спланируй архитектуру системы» → записал
2. «Сделай парсер тендеров с сайта» → записал
3. «Добавь AI-обработку текста» → записал
4. «Создай дашборд со статистикой» → записал

Один шаг = один чат. Результат каждого шага → в файл.

---

## Структура для больших проектов

```
my-project/
├── README.md           # Cursor читает первым
├── plan.md             # Общий план
├── progress.md         # Что уже сделано
├── .cursor/rules/
│   └── project.mdc     # Правила для AI
├── tasks/
│   ├── todo.md         # Что осталось
│   └── lessons.md      # Уроки из ошибок
└── src/                # Код
```

## Практический пример

> Андрей Погорелый делал систему анализа тендеров **полтора месяца**.
> Не одним запросом — а шаг за шагом, каждый раз записывая результат.

### День 1
```
Спланируй архитектуру: нужно парсить тендеры с 3 площадок,
анализировать AI, показывать в дашборде. Сохрани план в plan.md
```

### День 2
```
Прочитай plan.md. Начни с шага 1 — парсер для zakupki.gov.ru.
Запиши результат в progress.md
```

### День 3
```
Прочитай plan.md и progress.md. Продолжай с шага 2.
```

И так далее — каждый день новая сессия, но AI всегда в курсе.

---

## Чеклист для больших проектов

- [ ] README.md написан подробно
- [ ] plan.md с пошаговым планом
- [ ] progress.md обновляется после каждого шага
- [ ] Каждый шаг — отдельный чат
- [ ] git commit после каждого рабочего этапа
- [ ] .mdc правила для повторяющихся паттернов
- [ ] lessons.md для ошибок (чтобы AI не повторял)
