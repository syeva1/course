# Практика — Модуль 07. Вайб-кодинг

## Задание 1. Лендинг по README (повтори по инструкции)

1. Создай папку проекта, напиши README.md (описание, дизайн, секции)
2. Через Plan Mode составь план из 6-8 шагов
3. Пройди первые 3 шага (структура + стили + героя-секция)
4. Commit каждый шаг
5. Открой в браузере — проверь результат

Результат: Начало лендинга — работает, открывается, выглядит прилично.

---

## Задание 2. Доделай лендинг + адаптив (повтори по инструкции)

Продолжи проект из задания 1:

1. Пройди оставшиеся шаги плана (услуги, отзывы, форма, футер)
2. Проверь адаптив: F12 → иконка телефона → iPhone 14
3. Если что-то кривое — уточни с конкретными размерами (px, %)
4. Commit готовый лендинг

Результат: Полный лендинг, адаптивный, все секции на месте.

---

## Задание 3. Калькулятор стоимости (сделай сам)

Создай калькулятор для своего бизнеса. README напиши сам:
- Какие параметры вводит клиент
- Как считается цена (формула)
- Как должен выглядеть

Пример: калькулятор стоимости подбора ассистента.
- Ввод: тип ассистента (личный/бизнес/удалённый), количество часов в неделю, срочность
- Вывод: стоимость подбора + ежемесячная зарплата ассистента

Требования:
1. Работает в браузере (HTML/CSS/JS)
2. Адаптивный
3. Красивый (не голая форма на белом фоне)
4. Кнопка «Рассчитать» + результат

Результат: Рабочий калькулятор. Можно показать клиенту.

---

## Задание 4. Деплой на GitHub Pages (сделай сам)

Задеплой любой из своих проектов (лендинг или калькулятор):

1. Залей на GitHub
2. Settings → Pages → Deploy from branch → main
3. Подожди 1-2 минуты
4. Открой ссылку https://твой-username.github.io/название-репо/
5. Проверь на телефоне (открой ссылку в Telegram себе)

Результат: Твой продукт в интернете. Можешь скинуть ссылку кому угодно.

---

## Задание 5. Свой продукт с нуля (творческое)

Придумай и создай продукт для своего бизнеса. Варианты:
- **Квиз** — опросник для сбора заявок (5-7 вопросов → форма контактов)
- **Портфолио** — сайт с примерами работ и описанием услуг
- **Дашборд** — панель с метриками (пусть пока с фейковыми данными)
- **Chrome-расширение** — плагин для браузера (например, быстрые шаблоны ответов)
- **Мини-CRM** — таблица клиентов с фильтрами и поиском

Обязательно:
1. README.md с подробным описанием
2. План в plan.md
3. Прогресс в progress.md
4. Commit каждый шаг
5. Деплой (GitHub Pages или VPS)

Результат: Реальный продукт, задеплоенный и работающий. Это уже портфолио.
