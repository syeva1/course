# Инструкция — Модуль 06. API и интеграции

## Шаг 1. Что такое API

API — это способ для программ общаться друг с другом. Когда ты пишешь боту в Telegram — твоё приложение отправляет запрос через Telegram API. Когда ChatGPT отвечает — он делает это через OpenAI API.

Аналогия: API — это официант в ресторане. Ты (программа) говоришь официанту (API) что хочешь, он несёт заказ на кухню (сервер), получает блюдо (ответ) и приносит тебе.

Что нужно для работы с API:
- **URL** — адрес куда отправлять запрос (например, `https://api.anthropic.com/v1/messages`)
- **Ключ (API Key)** — твой пропуск (чтобы сервер знал кто ты)
- **Запрос (Request)** — что ты хочешь (текст, параметры)
- **Ответ (Response)** — что сервер вернул (данные, текст, ошибка)

Формат данных: почти все API используют **JSON** — текст с фигурными скобками:
```json
{
  "model": "claude-sonnet-4-20250514",
  "messages": [{"role": "user", "content": "Привет!"}]
}
```

Результат: Ты понимаешь что такое API и из чего состоит запрос.

## Шаг 3. Получи API-ключ Claude

1. Открой console.anthropic.com
2. Зарегистрируйся (email + подтверждение)
3. Зайди в Settings → API Keys
4. Нажми Create Key
5. Скопируй ключ (начинается с `sk-ant-`)
6. Вставь в `.env`: `ANTHROPIC_API_KEY=sk-ant-...`
7. Пополни баланс: от $5 (хватит на ~500 запросов Claude Sonnet)

Оплата из России: через @PlatiRublemRu_Bot в Telegram (покупаешь подарочную карту) или через VPN + иностранную карту.

Результат: Ключ получен и сохранён в .env.

## Шаг 4. Первый запрос к Claude API

Создай файл `test_claude.py`:

```python
import anthropic
from dotenv import load_dotenv

load_dotenv()

client = anthropic.Anthropic()

message = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "Привет! Расскажи о себе в 3 предложениях."}
    ]
)

print(message.content[0].text)
```

Запусти:
```bash
python test_claude.py
```

Если увидел ответ от Claude — API работает.

Попробуй изменить промпт на что-то полезное для бизнеса:
```python
"content": "Напиши 3 варианта заголовка для рекламы HR-агентства"
```

Частая ошибка: `AuthenticationError`. Проверь что ключ в .env правильный и баланс пополнен.

Результат: Ты отправил запрос к Claude API и получил ответ.

## Шаг 7. Telegram Bot API — отправка уведомлений

Если у тебя есть бот из Модуля 05 — используй его токен. Если нет — создай нового в @BotFather.

Узнай свой CHAT_ID:
1. Напиши что-нибудь боту в Telegram
2. Открой в браузере: `https://api.telegram.org/bot{ТВОЙ_ТОКЕН}/getUpdates`
3. Найди `"chat":{"id":123456789}` — это твой CHAT_ID
4. Добавь в `.env`: `TELEGRAM_CHAT_ID=123456789`

Создай `test_telegram.py`:

```python
import requests
import os
from dotenv import load_dotenv

load_dotenv()

TOKEN = os.getenv("TELEGRAM_BOT_TOKEN")
CHAT_ID = os.getenv("TELEGRAM_CHAT_ID")

url = f"https://api.telegram.org/bot{TOKEN}/sendMessage"
response = requests.post(url, json={
    "chat_id": CHAT_ID,
    "text": "Привет! Это тестовое уведомление от моего скрипта."
})

print("Отправлено!" if response.ok else f"Ошибка: {response.text}")
```

Запусти: `python test_telegram.py`

Проверь Telegram — должно прийти сообщение от бота.

Результат: Ты умеешь отправлять сообщения в Telegram из кода.

## Шаг 8. Google Sheets API (опционально)

Это сложнее, но очень полезно для бизнеса — автоматическая запись в таблицы.

1. Зайди на console.cloud.google.com
2. Создай новый проект (любое название)
3. В поиске найди «Google Sheets API» → Enable
4. Найди «Google Drive API» → Enable
5. Зайди в IAM & Admin → Service Accounts → Create Service Account
6. Заполни имя → Create → Done
7. Кликни на созданный аккаунт → Keys → Add Key → Create new key → JSON
8. Скачается файл `credentials.json` — положи в папку проекта
9. Открой JSON — найди строку `"client_email": "...@...iam.gserviceaccount.com"`
10. Создай Google Таблицу → Поделиться → вставь этот email → Редактор

Создай `test_sheets.py`:

```python
import gspread

gc = gspread.service_account(filename="credentials.json")
sh = gc.open("Название твоей таблицы")
ws = sh.sheet1

# Записать строку
ws.append_row(["Тест", "2026-03-10", "Работает!"])

# Прочитать все данные
data = ws.get_all_records()
print(data)
```

Результат: Ты записываешь и читаешь данные из Google Sheets через код.

## Шаг 10. MCP — подключение сервисов к Cursor

MCP (Model Context Protocol) — способ подключить внешние сервисы прямо в Cursor. После подключения AI может сам обращаться к этим сервисам.

Например: подключаешь GitHub MCP → AI может создавать issues, PR, читать код из репозитория.

Как подключить:
1. Создай файл `.cursor/mcp.json` в проекте
2. Добавь конфигурацию:

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_..."
      }
    }
  }
}
```

3. Перезапусти Cursor: Ctrl+Shift+P → Reload Window
4. В Agent Mode теперь доступны инструменты GitHub

Подробные примеры — в `материалы/MCP_подключение.md` и `материалы/гайд_MCP.md`.

Результат: Ты понимаешь что такое MCP и как подключить внешний сервис к Cursor.

## Шаг 11. No-code альтернативы

Не всё нужно писать кодом. Для простых связок есть no-code инструменты:

- **Zapier** (zapier.com) — 5 бесплатных сценариев, 100+ сервисов
- **Make** (make.com) — бесплатный план, мощнее Zapier
- **n8n** (n8n.io) — бесплатный self-hosted, полная свобода
- **Albato** (albato.ru) — российский аналог, рублёвая оплата

Когда код, когда no-code:
- No-code: простые связки (новый email → уведомление в Telegram)
- Код: сложная логика, обработка данных, AI, кастомизация

Результат: Ты знаешь альтернативы и умеешь выбирать подход.

## Шаг 12. Безопасность API-ключей

Правила, которые нельзя нарушать:

1. Ключи ТОЛЬКО в .env, НИКОГДА в коде
2. .env ВСЕГДА в .gitignore
3. Если ключ попал на GitHub — немедленно его отзови и создай новый
4. Каждый сервис → свой ключ (не используй один ключ везде)
5. Ограничивай права: если ключу нужен только read — не давай write

Как проверить что .env не попал на GitHub:
```bash
git log --all --full-history -- .env
```

Если показывает результаты — .env был в истории. Нужно очистить историю или создать новые ключи.

Результат: Ты понимаешь правила безопасности API-ключей.
