# Инструкция — Модуль 07: AI-агенты

## Зачем мне это?

Ты каждый день пишешь одни и те же промпты: «напиши пост», «ответь клиенту», «проанализируй отчёт». Каждый раз объясняешь AI, кто ты, что за бизнес, какой стиль. Агент — это способ объяснить один раз и получать результат каждый раз.

---

## Шаг 1. Поймите разницу: промпт vs агент

### ❌ Промпт (одноразовый запрос)

Ты пишешь в ChatGPT:
```
Напиши пост для Telegram про подбор ассистентов.
Тон — экспертный, аудитория — предприниматели.
Не пиши канцелярит. Добавь примеры.
```

Результат: нормальный текст, но в следующий раз ты снова будешь писать всё это.

### ✅ Агент-копирайтер (постоянный сотрудник)

Ты создаёшь файл с правилами ОДИН РАЗ. Потом пишешь просто:
```
Напиши пост про подбор ассистентов
```

Агент УЖЕ знает: бренд АССИСТ+, аудиторию, стиль, примеры лучших постов. Результат — в 10 раз лучше, и каждый раз стабильный.

---

## Шаг 2. Определите роль агента

Ответьте на четыре вопроса:

1. **Роль** — кто этот агент? (копирайтер, ассистент, аналитик)
2. **Контекст** — что он знает? (бренд, цены, ЦА, примеры)
3. **Задачи** — что он делает? (пишет посты, отвечает клиентам, анализирует данные)
4. **Правила** — как себя ведёт? (тон, формат, чего НЕ делать)

### Пример для агента-копирайтера АССИСТ+

| Вопрос | Ответ |
|--------|-------|
| Роль | Копирайтер бренда АССИСТ+ |
| Контекст | HR-агентство, подбор ассистентов, цены от 30К, ЦА — предприниматели 25-45 |
| Задачи | Посты для Telegram, тексты для лендинга, email-рассылки |
| Правила | Экспертный живой тон, без канцелярита, на «ты», с примерами из практики |

## Шаг 3. Создайте файл конфигурации

### Вариант А: Cursor Rules (рекомендуется)

Создайте файл `.cursor/rules/copywriter.mdc`:

```markdown
---
description: "Копирайтер бренда АССИСТ+"
globs: ["**/*"]
---

## Роль
Ты — копирайтер компании АССИСТ+.

## Контекст
- АССИСТ+ — HR-агентство по подбору бизнес-ассистентов для предпринимателей
- Аудитория: предприниматели 25–45 лет, у которых нет времени на рутину
- Цены: подбор от 30 000 ₽, ассистент на аутсорсе от 25 000 ₽/мес
- Ценности бренда: скорость (подбор за 3–5 дней), качество, забота

## Задачи
- Писать посты для Telegram-канала (1000–1500 символов)
- Создавать тексты для лендингов
- Писать email-рассылки

## Правила
- Тон: экспертный, живой, без канцелярита
- Обращение: на «ты»
- Формат: короткие абзацы, с примерами из практики
- НЕ пиши: «Уважаемые коллеги», «Рады сообщить», «В рамках»
- Примеры хорошего тона: «Нанял ассистента — освободил 3 часа в день»
- Примеры плохого тона: «Мы предоставляем услуги по подбору персонала»

## Примеры
Хороший пост: «Знаешь, что общего между успешными предпринимателями? Они не делают всё сами. Ассистент — это не роскошь, а инструмент роста.»
```

### Вариант Б: CLAUDE.md

## Шаг 4. Скиллы — модули для агента

Скиллы — это папки с инструкциями для конкретных задач. Агент читает их, когда нужно.

### Пример: скилл «лендинг»

```
skills/landing/
├── SKILL.md              # Инструкции: как создавать лендинг
├── templates/
│   └── structure.md       # Шаблон структуры: заголовок → боль → решение → цена → CTA
└── examples/
    └── good_landing.md    # Пример хорошего лендинга
```

### Как это работает на практике

1. Ты пишешь агенту: «Сделай лендинг для нового курса»
2. Агент читает `skills/landing/SKILL.md`
3. Берёт шаблон структуры из `templates/`
4. Смотрит пример из `examples/`
5. Генерирует лендинг, который учитывает твой бренд, ЦА и стиль

**Без скилла:** шаблонная страница → переписываешь 80%
**Со скиллом:** готовая страница → правишь 10%

## Шаг 5. Тестируйте и улучшайте

1. Дайте агенту 5 реальных задач
2. Оцените каждый результат: 1–5 баллов
3. Запишите ошибки в `lessons.md`
4. Обновите конфигурацию — добавьте правила для исправления
5. Повторите — качество растёт с каждой итерацией

### Пример lessons.md

```markdown
# Lessons

## 2026-03-09
- Агент использует «Уважаемые коллеги» → добавил в правила: «НЕ пиши»
- Посты слишком длинные → добавил лимит: 1000–1500 символов
- Не добавляет CTA → добавил правило: каждый пост заканчивается призывом к действию
```

## Шаг 6. Создайте команду из нескольких агентов

Когда один агент работает стабильно — создайте следующего:

| Агент | Файл | Что делает |
|-------|------|-----------|
| Копирайтер | `.cursor/rules/copywriter.mdc` | Пишет посты и тексты |
| Ассистент | `.cursor/rules/assistant.mdc` | Отвечает на вопросы клиентов |
| Аналитик | `.cursor/rules/analyst.mdc` | Анализирует данные, делает отчёты |
| HR | `.cursor/rules/hr.mdc` | Пишет вакансии, скринит резюме |

**Каждый агент — отдельный файл.** Cursor сам подберёт нужного по контексту задачи.
