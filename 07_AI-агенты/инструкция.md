# Инструкция — Модуль 07: AI-агенты

## Шаг 1. Поймите разницу: промпт vs агент

**Промпт** — это одноразовый запрос. Вы каждый раз объясняете контекст заново.

**Агент** — это конфигурация, которая хранится в файле. Он знает свою роль, контекст, задачи и правила. Работает стабильно каждый раз.

## Шаг 2. Определите роль агента

Ответьте на четыре вопроса:

1. **Роль** — кто этот агент? (ассистент, аналитик, копирайтер)
2. **Контекст** — что он знает? (файлы, данные, история)
3. **Задачи** — что он делает? (отвечает, анализирует, пишет)
4. **Правила** — как себя ведёт? (тон, формат, ограничения)

## Шаг 3. Создайте файл конфигурации

### Вариант А: Cursor Rules

Создайте файл `.cursor/rules/agent-name.mdc`:

```markdown
---
description: "Описание агента"
globs: ["**/*"]
---

## Роль
Ты — [роль] компании [компания].

## Контекст
- Знаешь: [что знает]
- Имеешь доступ к: [файлы, данные]

## Задачи
- [задача 1]
- [задача 2]

## Правила
- [правило 1]
- [правило 2]
```

### Вариант Б: CLAUDE.md

Создайте `CLAUDE.md` в корне проекта:

```markdown
# CLAUDE.md

## Роль
[Описание роли]

## Контекст проекта
[Что агент должен знать]

## Правила работы
[Как себя вести]

## Границы
[Что нельзя делать]
```

## Шаг 4. Используйте скиллы

Скиллы — это переиспользуемые модули для агента:

```
skills/
├── seo-optimizer/
│   ├── SKILL.md
│   └── templates/
├── landing-generator/
│   ├── SKILL.md
│   └── scripts/
└── email-writer/
    ├── SKILL.md
    └── examples/
```

Каждый скилл содержит `SKILL.md` с инструкциями, которые агент читает при необходимости.

## Шаг 5. Тестируйте и улучшайте

1. Дайте агенту 5 реальных задач
2. Оцените результаты
3. Запишите ошибки в `lessons.md`
4. Обновите конфигурацию агента
5. Повторите

### Пример lessons.md

```markdown
# Lessons

## 2026-03-09
- Агент использует слишком формальный тон → добавил правило "пиши живо"
- Не учитывает ЦА → добавил описание целевой аудитории в контекст
- Слишком длинные ответы → добавил лимит "не более 200 слов"
```

## Шаг 6. Масштабируйте

Когда один агент работает стабильно — создайте следующего. Постепенно соберите команду:

- **Агент-ассистент** — отвечает клиентам
- **Агент-аналитик** — анализирует данные
- **Агент-копирайтер** — пишет контент
- **Агент-HR** — помогает с подбором
