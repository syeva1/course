# Шпаргалка — Модуль 07: AI-агенты

## Формула агента

```
Агент = Роль + Контекст + Задачи + Правила
```

## Промпт vs Агент

| | Промпт | Агент |
|---|--------|-------|
| Память | Нет | Да (файл конфигурации) |
| Стабильность | Зависит от формулировки | Стабильный результат |
| Переиспользование | Копипаст | Git + скиллы |
| Улучшение | Каждый раз заново | lessons.md |

## Структура конфигурации

```markdown
## Роль
Кто агент и для какой компании

## Контекст
Что агент знает (файлы, данные, история)

## Задачи
Что агент делает (список задач)

## Правила
Как себя ведёт (тон, формат, границы)

## Примеры
Образцы хороших ответов
```

## Скилл — структура

```
skills/название/
├── SKILL.md          # Инструкции для агента
├── templates/        # Шаблоны
├── scripts/          # Скрипты
└── examples/         # Примеры
```

## Цикл улучшения

```
Задача → Результат → Оценка → lessons.md → Обновление → Повтор
```

## Cursor Rules — быстрый старт

Файл: `.cursor/rules/agent.mdc`

```
---
description: "Описание"
globs: ["**/*"]
---
[Конфигурация агента]
```

## CLAUDE.md — быстрый старт

Файл: `CLAUDE.md` в корне проекта

```markdown
# CLAUDE.md
[Конфигурация агента]
```

## 5 типов агентов для бизнеса

1. **Ассистент** — отвечает клиентам, квалифицирует лиды
2. **Копирайтер** — пишет контент в стиле бренда
3. **Аналитик** — анализирует данные, строит отчёты
4. **HR** — помогает с подбором и онбордингом
5. **Маркетолог** — SEO, лендинги, email-рассылки

## Частые ошибки

- ❌ Слишком общая роль
- ❌ Нет примеров ответов
- ❌ Нет границ (что нельзя делать)
- ❌ Один мега-агент вместо команды
- ❌ Не улучшаешь через lessons.md
- ❌ Нет контекста о компании
