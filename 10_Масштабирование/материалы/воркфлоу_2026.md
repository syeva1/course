# Продвинутые воркфлоу Cursor 2026

## 1. Research-First Protocol (5 фаз)

Для сложных задач — систематический подход:

1. **DISCOVERY** → Пойми текущее состояние
   - «@Codebase Покажи все места, где используется UserService. Не предлагай изменений.»
2. **PLAN** → Создай план (в файл `plan.md`)
3. **CRITIQUE** → Переключись на другую модель и проверь план
4. **EXECUTE** → Реализуй пошагово через Agent
5. **AUDIT** → Проверь через `git diff`, тесты, линтер

## 2. Spec-First Development

Вместо «сделай мне приложение» — напиши спецификацию:

```
project/
├── product_spec.md    # Что приложение делает
├── tech_spec.md       # Как построено технически
├── .cursor/rules/
│   └── architect.mdc  # Правило: читай спеки перед кодом
└── src/
```

AI читает спеки и строит по ним. Результат в 5 раз точнее.

## 3. Lovable + Cursor (Vibe Coding → Production)

1. **Lovable** → Нарисуй UI визуально, подключи к GitHub
2. **Clone** → Скачай репо локально
3. **Cursor** → Добавь логику, API, сложные фичи
4. **Push** → Синхронизируй обратно

Люди создают полные SaaS-продукты за 4 дня этим методом.

## 4. Worktrees — параллельные агенты

Запусти несколько агентов одновременно через git worktrees:

```bash
git worktree add ../my-project-refactor refactor-branch
git worktree add ../my-project-tests test-branch
```

- Агент 1: Рефакторинг компонента
- Агент 2: Написание тестов
- Нет конфликтов файлов!

## 5. PlayWhite (Playwright + MCP)

Автотесты на автопилоте:
1. Подключи Playwright MCP
2. Попроси: «Проверь форму логина на localhost:3000»
3. Cursor запустит браузер → Найдёт баг → Исправит код → Перепроверит

## 6. Cursor 2.4 — Subagents

Главный агент разбивает задачу на подзадачи:
```
Главный агент
├── Terminal Subagent → Запускает команды
├── Docs Subagent → Читает документацию
├── Test Subagent → Пишет и запускает тесты
└── Refactor Subagent → Меняет код
```

Параллельная обработка — в разы быстрее для сложных фич.
