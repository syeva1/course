# Инструкция — Telegram-бот за 10 шагов

## Зачем мне бот?

Представь: ты нашёл классную статью, надиктовал мысль голосовым, получил PDF от клиента. Всё это в Telegram. А работаешь ты в Cursor. **Бот — мост между Telegram и Cursor.** Скинул в бота → оно уже в твоём проекте как контекст.

---

## Шаг 1. Создайте бота через @BotFather

Откройте Telegram, найдите @BotFather и отправьте `/newbot`. Придумайте имя и username (должен заканчиваться на `bot`). Скопируйте токен.

**Что получишь:** Бот уже существует в Telegram. Пока пустой, но его можно найти по username.

## Шаг 2. Поймите, что такое токен

**Токен — как пароль от бота.** Кто его знает — управляет ботом. Потерял или опубликовал в открытом коде — любой может перехватить бота.

- Храни токен ТОЛЬКО в файле `.env`
- Никогда не пиши токен в код напрямую
- Если утёк — перевыпусти через @BotFather → `/revoke`

## Шаг 3. Создайте файл .env

```
TELEGRAM_TOKEN=ваш_токен_от_BotFather
OPENAI_API_KEY=ваш_ключ_openai
```

Добавьте `.env` в `.gitignore` — иначе токен попадёт на GitHub.

**Что получишь:** Безопасное хранение паролей. Код можно показывать кому угодно.

## Шаг 4. Установите Python и библиотеки

```bash
pip install python-telegram-bot openai python-dotenv
```

## Шаг 5. Напишите echo-бота

Скопируйте код из `материалы/код_echo_бота.py`. Запустите:

```bash
python код_echo_бота.py
```

Напишите боту в Telegram — он должен ответить тем же текстом.

**Что получишь:** Рабочий бот. Написал «привет» — получил «привет». 12 строк кода — и он уже живой!

## Шаг 6. Поймите, как работают обработчики

**Обработчик — как секретарь.** У тебя в офисе сидят несколько секретарей: один принимает звонки, другой — почту, третий — посетителей. Обработчики бота работают так же:

- 📝 Пришёл **текст** → обработчик сохраняет как .md файл
- 🎤 Пришло **голосовое** → обработчик транскрибирует в текст
- 📎 Пришёл **файл** → обработчик сохраняет в нужную папку
- 🖼 Пришла **картинка** → обработчик сохраняет с датой

Каждый обработчик — отдельная функция в коде. Cursor напишет их за тебя.

## Шаг 7. Подключите AI

Скопируйте код из `материалы/код_ai_бота.py`. Замените токены на свои. Теперь бот отвечает с помощью нейросети.

**Что получишь:** Бот, который отвечает осмысленно. Спросил «Какие у вас цены?» — получил развёрнутый ответ.

## Шаг 8. Напишите системный промпт

Опишите, кто ваш бот, что он знает о бизнесе, как должен отвечать. Чем точнее промпт — тем полезнее бот.

**Пример:**
```
Ты — ассистент компании АССИСТ+.
Мы подбираем бизнес-ассистентов для предпринимателей.
Цены: от 30 000 ₽/мес. Срок подбора: 3–5 дней.
Если клиент хочет оставить заявку — попроси имя и телефон.
Отвечай дружелюбно и по делу.
```

**Что получишь:** Бот, который знает всё о бизнесе и отвечает клиентам вместо тебя.

## Шаг 9. Добавьте кнопки (InlineKeyboard)

```python
from telegram import InlineKeyboardButton, InlineKeyboardMarkup

keyboard = InlineKeyboardMarkup([
    [InlineKeyboardButton("📋 Услуги", callback_data="services")],
    [InlineKeyboardButton("💰 Цены", callback_data="prices")],
    [InlineKeyboardButton("📞 Связаться", callback_data="contact")],
])
```

Обработайте нажатия через `CallbackQueryHandler`.

**Что получишь:** Красивый бот с кнопками, как у профессиональных сервисов.

## Шаг 10. Задеплойте бота

**Бот работает только когда комп включён.** Закрыл ноутбук — бот «уснул». Хочешь 24/7 — нужен сервер. Это как квартира для бота: он там живёт и работает, пока ты спишь.

**Варианты:**
- **Railway** (бесплатно для старта) — самый простой вариант
- **VPS** (от 300 ₽/мес) — самый надёжный
- **Render** (бесплатно, но бот «засыпает» через 15 мин)

```bash
# На VPS:
nohup python bot.py &
```

**Что получишь:** Бот отвечает клиентам 24/7, даже ночью, пока ты спишь.
