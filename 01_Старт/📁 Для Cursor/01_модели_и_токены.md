## Что такое модель

Cursor — это просто блокнот. Умный блокнот, но блокнот. Думает не он, а **модель** — внешний мозг на сервере.

Когда ты пишешь запрос → Cursor отправляет его на сервер → модель обрабатывает → возвращает ответ → Cursor применяет изменения.

## Какие модели бывают

| Компания | Модели | Для чего лучше |
|----------|--------|---------------|
| **Anthropic** | Claude Sonnet, Opus | Код, длинные задачи, рефакторинг |
| **OpenAI** | GPT-5, Codex | Код, универсальные задачи |
| **Google** | Gemini | Анализ данных, длинные документы |
| **Cursor** | Cursor Small | Быстрые простые задачи |

### Режим Auto
Оставь **Auto** — Cursor сам выберет модель. Для простых задач возьмёт дешёвую, для сложных — мощную. На старте этого достаточно.

## Что такое токены

Токен ≈ одно слово. 

Ты отправил запрос на 500 слов + файл на 5000 слов = ~5500 токенов на входе.
Модель ответила на 1000 слов = ~1000 токенов на выходе.
Итого: ~6500 токенов за один обмен.

## Контекстное окно

Это «оперативная память» модели — максимум токенов за одну сессию.

Когда окно заполняется → модель **сжимает** историю → детали теряются.

> Знакомая ситуация: работали с AI час, а потом он «забыл» о чём договорились в начале. Он не сломался — просто сжал контекст.

### Что делать:
1. **Одна задача = один чат** (не мешай всё в одну сессию)
2. **Результаты в файл** (чтобы в новом чате дать AI контекст)
3. **Промпты короче** (меньше воды = больше места для полезного)
