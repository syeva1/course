# Инструкция — Инструменты Cursor + GitHub

Пошаговый гайд по каждому инструменту. Делайте по порядку.

---

## 1. Tab — автодополнение

**Что это:** AI предсказывает, что вы напишете, и предлагает серый текст.

**Как использовать:**
1. Откройте любой файл в Cursor
2. Начните писать код или текст
3. Увидите серый текст — это предложение AI
4. **Tab** — принять предложение
5. **Esc** или продолжить печатать — отклонить

**Советы:**
- Напишите комментарий перед кодом — AI поймёт контекст лучше
- Чем больше написано в файле, тем точнее подсказки
- Работает для любого языка программирования

---

## 2. Cmd+K — Inline Edit

**Что это:** Быстрая AI-команда для редактирования выделенного кода.

**Как использовать:**
1. Выделите фрагмент кода
2. Нажмите **Cmd+K** (Windows: **Ctrl+K**)
3. Появится поле ввода — напишите, что нужно сделать
4. Примеры команд:
   - «Переведи комментарии на русский»
   - «Добавь обработку ошибок»
   - «Упрости этот код»
   - «Добавь типы TypeScript»
5. AI покажет изменения — примите или отклоните

**Советы:**
- Пишите конкретные инструкции
- Можно использовать без выделения — для генерации нового кода

---

## 3. Cmd+L — AI Chat

**Что это:** Чат с AI, который видит ваш проект.

**Как использовать:**
1. Нажмите **Cmd+L** (Windows: **Ctrl+L**)
2. Откроется панель чата справа
3. Задайте вопрос:
   - «Что делает функция на строке 15?»
   - «Почему этот код не работает?»
   - «Как добавить авторизацию?»
4. Прикрепите файлы: напишите **@имяфайла**
5. AI ответит с учётом контекста вашего проекта

**Советы:**
- Используйте для понимания чужого кода
- Хорошо работает для отладки ошибок
- Можно вести диалог — уточнять и задавать follow-up

---

## 4. Cmd+I — Composer

**Что это:** Мощный режим, где AI работает с несколькими файлами.

**Как использовать:**
1. Нажмите **Cmd+I** (Windows: **Ctrl+I**)
2. Откроется Composer
3. Опишите задачу целиком:
   - «Создай landing page с формой обратной связи»
   - «Добавь страницу "О нас" в проект»
   - «Рефактори все компоненты на TypeScript»
4. AI создаст/изменит несколько файлов
5. Просмотрите diff каждого файла
6. Примите или отклоните изменения

**Советы:**
- Описывайте задачу максимально подробно
- Используйте для больших задач
- Проверяйте результат перед принятием

## 5. GitHub — подключение и работа

### Создание аккаунта
1. Откройте [github.com](https://github.com)
2. Нажмите **Sign up**
3. Введите email, username, пароль
4. Подтвердите email
5. Выберите бесплатный план (Free)

### Подключение к Cursor
1. Откройте проект в Cursor
2. Нажмите иконку **Source Control** (ветка) в боковой панели
3. Нажмите **Initialize Repository**
4. Cursor предложит войти в GitHub — нажмите **Allow**
5. Авторизуйтесь в браузере

### Первый commit и push
1. Измените файл
2. В Source Control увидите файл в списке изменений
3. Нажмите **+** напротив файла (Stage)
4. Введите сообщение коммита (например: «Добавил главную страницу»)
5. Нажмите **✓ Commit**
6. Нажмите **Publish Branch**
7. Выберите **Public** или **Private** репозиторий
8. Готово! Проект на GitHub

### Pull — получение изменений
1. Нажмите **…** в Source Control → **Pull**
2. Или используйте кнопку синхронизации (↻)
