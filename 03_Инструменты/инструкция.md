# Инструкция — Модуль 03. Инструменты Cursor + GitHub

## Шаг 1. Tab — автодополнение

Что это: AI предсказывает что ты напишешь и предлагает серый текст. Нажимаешь Tab — текст вставляется.

Где найти: работает автоматически в любом файле. Ничего включать не нужно.

Как попробовать:
1. Открой любой файл в проекте (или создай новый — Cmd+N)
2. Начни писать что-нибудь. Например, в HTML-файле набери `<div class="`
3. Появится серый текст — это предложение AI
4. Tab — принять предложение
5. Esc — отклонить
6. Просто продолжи печатать — предложение исчезнет

Как сделать подсказки точнее:
- Напиши комментарий перед кодом. Например: `// Кнопка с красным фоном`. Потом начни писать — Tab предложит именно красную кнопку
- Чем больше контекста в файле, тем умнее подсказки
- Работает для любого языка: HTML, CSS, JavaScript, Python, текст

Частая проблема: Tab вставляет ерунду. Решение: не принимай всё подряд. Прочитай серый текст. Если не то — Esc и пиши сам.

Горячие клавиши:
- Tab — принять
- Esc — отклонить
- Alt+] — следующее предложение (если есть варианты)

Результат: Ты видишь серые подсказки при наборе и умеешь их принимать/отклонять.

## Шаг 2. Cmd+K — быстрое редактирование

Что это: выделяешь кусок текста или кода, нажимаешь Cmd+K, пишешь что сделать — AI редактирует.

Где найти: работает в любом файле. Горячая клавиша Cmd+K (Mac) или Ctrl+K (Windows).

Как попробовать:
1. Открой файл index.html из Модуля 01
2. Выдели заголовок `<h1>` (или любой фрагмент)
3. Нажми Cmd+K (Ctrl+K на Windows)
4. Появится поле ввода сверху. Напиши: «Сделай заголовок крупнее и добавь подзаголовок»
5. AI покажет изменения — зелёное = добавлено, красное = удалено
6. Нажми Accept — изменения применятся
7. Reject — отменятся

Примеры команд для Cmd+K:
- «Переведи это на русский»
- «Добавь обработку ошибок»
- «Упрости этот код»
- «Оберни в функцию»
- «Добавь комментарии»

Без выделения: если ничего не выделять и нажать Cmd+K, AI сгенерирует новый код в текущей позиции курсора.

Горячие клавиши:
- Cmd+K / Ctrl+K — открыть
- Enter — отправить команду
- Cmd+Shift+K — принять изменения
- Esc — отменить

Результат: Ты умеешь редактировать выделенный код одной командой.

## Шаг 3. Cmd+L — AI Chat

Что это: чат с AI, который видит твой проект. Задаёшь вопросы, AI отвечает. Можно подключать файлы через @.

Где найти: правая панель. Горячая клавиша Cmd+L (Mac) или Ctrl+L (Windows).

Как попробовать:
1. Нажми Cmd+L — откроется панель чата справа
2. Напиши: «Объясни что делает файл index.html в этом проекте»
3. AI ответит, проанализировав файл
4. Задай уточняющий вопрос: «Как добавить туда форму обратной связи?»
5. AI ответит с учётом предыдущего контекста

Как подключать файлы:
- Набери `@` — появится список
- @File — конкретный файл
- @Codebase — весь проект
- @Web — интернет
- Подробнее про @-символы — в Модуле 02

Когда использовать Chat:
- Понять чужой код
- Отладить ошибку
- Спросить совет
- Попросить написать код (который потом скопируешь)

Когда НЕ использовать Chat:
- Для изменения файлов (лучше Cmd+K или Composer)
- Chat показывает код в ответе, но не применяет его автоматически

Горячие клавиши:
- Cmd+L / Ctrl+L — открыть чат
- Cmd+Shift+L / Ctrl+Shift+L — добавить выделенный код в чат

Результат: Ты ведёшь диалог с AI и получаешь ответы с учётом проекта.

## Шаг 4. Cmd+I — Composer (мультифайловый режим)

Что это: самый мощный инструмент. AI может создавать и редактировать несколько файлов одновременно.

Где найти: горячая клавиша Cmd+I (Mac) или Ctrl+I (Windows). Или через Agent Side Bar (левая панель, иконка AI).

Как попробовать:
1. Нажми Cmd+I — откроется Composer
2. Напиши: «Создай страницу about.html — "О компании". Используй такой же стиль как в index.html. Добавь навигацию между страницами на обеих страницах.»
3. AI создаст/изменит несколько файлов
4. Для каждого файла покажет diff (что добавлено, что удалено)
5. Просмотри изменения
6. Accept All — принять всё, или Accept/Reject по файлам

Три режима Composer:
- **Ask** — AI отвечает текстом, ничего не меняет (как Chat)
- **Agent** — AI сам решает какие файлы создать/изменить, может запускать терминал
- **Plan** — AI сначала составляет план, показывает его, и только после твоего одобрения выполняет

Как переключить режим: в нижней части Composer есть выпадающий список (Ask / Agent / Plan).

Когда какой режим:
- Ask — когда хочешь спросить, не меняя ничего
- Agent — для средних задач (добавить страницу, исправить ошибку)
- Plan — для больших задач (новая фича, рефакторинг, создание проекта)

Горячие клавиши:
- Cmd+I / Ctrl+I — открыть Composer
- Cmd+Enter — отправить в режиме Agent

Результат: Ты создал новый файл через Composer и видишь как AI работает с несколькими файлами.

## Шаг 5. Plan Mode — подробнее

Plan Mode заслуживает отдельного шага, потому что это самый правильный способ работать с большими задачами.

Как работает:
1. Переключи Composer в режим Plan
2. Опиши задачу подробно
3. AI составит пошаговый план: какие файлы создать, что в них написать, в каком порядке
4. Ты читаешь план. Если что-то не так — пишешь комментарий, AI корректирует
5. Когда план одобрен — AI выполняет его шаг за шагом
6. После каждого шага ты можешь остановиться и проверить

Зачем это нужно:
- AI не уйдёт не туда — ты контролируешь направление ДО того как что-то создано
- Большие задачи разбиваются на маленькие шаги
- Легче отменить одну часть, чем переделывать всё

Пример промпта для Plan Mode:
```
Мне нужен лендинг для HR-агентства. 
Секции: герой с заголовком и кнопкой, 3 преимущества, отзывы клиентов, форма заявки, футер.
Стиль: профессиональный, красный акцент #E63229.
Файлы: index.html, css/styles.css, js/main.js.
Составь план реализации.
```

Результат: Ты составил план через Plan Mode и выполнил его пошагово.

## Шаг 6. «AI сам напишет запрос»

Важная техника: не пиши промпт сам — попроси AI написать промпт за тебя.

Как это работает:
1. Открой Chat (Cmd+L)
2. Напиши: «Я хочу создать калькулятор стоимости услуг для моего бизнеса. Напиши подробный промпт для Composer, который создаст это максимально качественно.»
3. AI напишет детальный промпт
4. Скопируй его и вставь в Composer (Cmd+I)

Зачем: AI знает, какие детали нужны для хорошего результата. Его промпт будет детальнее и точнее, чем тот что ты напишешь сам.

Результат: Ты получаешь лучший результат, потому что промпт написан профессионально.

## Шаг 7. Терминал в Cursor

Что это: встроенный терминал (командная строка) прямо внутри Cursor.

Где найти: внизу окна Cursor. Горячая клавиша Cmd+` (Mac) или Ctrl+` (Windows). Символ ` — это обратная кавычка, левее от цифры 1.

Как попробовать:
1. Нажми Cmd+` — откроется терминал
2. Набери `ls` (Mac/Linux) или `dir` (Windows) — увидишь список файлов проекта
3. Набери `pwd` (Mac/Linux) — увидишь путь к проекту

Зачем нужен:
- Установка пакетов (pip install, npm install)
- Запуск скриптов (python bot.py)
- Работа с Git (git status, git push)
- Проверка что файлы на месте

Базовые команды есть в `материалы/команды_терминала.md`.

Результат: Ты открываешь терминал и выполняешь базовые команды.

## Шаг 8. GitHub — создание аккаунта

GitHub — это место, где хранятся проекты. Как облако для кода.

1. Открой github.com
2. Нажми Sign up
3. Введи email, придумай username (латиницей), пароль
4. Подтверди email (придёт письмо)
5. Выбери бесплатный план (Free)

Username — это твоё имя на GitHub. Выбирай нормальное, оно будет видно всем. Например: `ivanova-maria`, а не `xXx_killer_xXx`.

Результат: У тебя есть аккаунт на github.com.

## Шаг 9. Git — подключение к проекту

1. В Cursor открой свой проект
2. В левой панели нажми иконку Source Control (ветка с точками, третья сверху)
3. Нажми Initialize Repository
4. Cursor создаст Git-репозиторий в папке проекта

Теперь подключи GitHub:
1. В Source Control нажми кнопку Publish Branch
2. Cursor предложит войти в GitHub — нажми Allow
3. Откроется браузер — авторизуйся
4. Выбери имя для репозитория
5. Выбери Public (видно всем) или Private (только тебе)
6. Нажми OK

Результат: Проект подключён к GitHub. Ты видишь его на github.com в своём профиле.

## Шаг 10. Commit и Push — сохранение изменений

Commit — это «сохранение» изменений с описанием что ты сделал.
Push — это отправка сохранённых изменений на GitHub.

1. Измени любой файл в проекте (добавь строчку, удали пробел — что угодно)
2. В Source Control увидишь файл в списке Changes
3. Нажми + напротив файла — он переместится в Staged Changes
4. В поле сверху напиши сообщение, например: «Добавил секцию отзывов»
5. Нажми галочку ✓ (Commit)
6. Нажми кнопку Sync Changes (или Push) — изменения уйдут на GitHub

Проверка: открой github.com, зайди в свой репозиторий — файлы обновились.

Правила хороших commit-сообщений:
- Пиши что сделал, а не что изменил: «Добавил форму заявки» > «Изменил index.html»
- Одна задача = один commit
- На русском нормально, на английском тоже

Горячие клавиши:
- Cmd+Shift+G / Ctrl+Shift+G — открыть Source Control
- Cmd+Enter — сделать commit (когда курсор в поле сообщения)

Результат: Ты сделал commit и push. Изменения видны на GitHub.

## Шаг 11. Расширения

Расширения добавляют новые возможности в Cursor.

Где найти: левая панель → иконка с квадратиками (Extensions). Или Cmd+Shift+X / Ctrl+Shift+X.

Как установить:
1. Открой Extensions
2. Введи название в поиск
3. Нажми Install
4. Некоторые расширения просят перезагрузить Cursor — нажми Reload

Рекомендуемые расширения:
- **Russian Language Pack** — русский интерфейс (если нужен)
- **Prettier** — автоформатирование кода при сохранении
- **GitLens** — показывает кто и когда менял каждую строку
- **Live Server** — открывает HTML-файлы с автообновлением

Как настроить Prettier:
1. Установи расширение
2. Зайди в Settings (Cmd+,)
3. Набери в поиске «format on save»
4. Поставь галочку Editor: Format On Save
5. Теперь при сохранении файла (Cmd+S) код автоматически форматируется

Результат: Расширения установлены, Prettier форматирует код при сохранении.

## Шаг 12. Итерация — главный паттерн

Весь этот модуль — про инструменты. Но главное — не сами инструменты, а паттерн работы:

1. Опиши задачу в Plan Mode
2. AI составит план — проверь его
3. AI выполнит — проверь результат
4. Если что-то не так — уточни через Chat или Cmd+K
5. Commit и Push когда готово

Маленькие шаги > большие задачи. Не проси AI сделать всё за раз. Разбивай на части.
