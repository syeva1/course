# 💻 Команды терминала — шпаргалка

> Терминал в Cursor — это командная строка прямо внутри редактора.
> Открыть: **Ctrl+`** (кнопка слева от "1" на клавиатуре)

---

## Как попасть в терминал

1. Открой Cursor
2. Нажми **Ctrl+`** (Mac: **Cmd+`**) — внизу появится панель терминала
3. Ты уже внутри папки проекта — можно писать команды

Или: меню **Terminal → New Terminal**

---

## Git: подключить папку к GitHub

### Первый раз — создаём репозиторий

```bash
# 1. Инициализация Git в папке
git init

# 2. Добавить все файлы
git add .

# 3. Первый коммит (сохранение)
git commit -m "первый коммит"

# 4. Подключить к репозиторию на GitHub
# (замени ТВОЙ_НИК и НАЗВАНИЕ_РЕПО на свои)
git remote add origin https://github.com/ТВОЙ_НИК/НАЗВАНИЕ_РЕПО.git

# 5. Отправить на GitHub
git push -u origin main
```

### Ежедневная работа

```bash
# Скачать свежие изменения
git pull

# Посмотреть, что изменилось
git status

# Добавить изменения
git add .

# Сохранить с описанием
git commit -m "описание что сделал"

# Отправить на GitHub
git push
```

## Токены и авторизация

При первом `git push` GitHub попросит авторизацию:

### Вариант 1: Через браузер (проще всего)
- GitHub сам откроет браузер → нажми "Authorize"
- Всё, дальше будет работать автоматически

### Вариант 2: Personal Access Token
1. Зайди на GitHub → **Settings → Developer Settings → Personal Access Tokens → Tokens (classic)**
2. Нажми **Generate new token**
3. Выбери scope: **repo** (галочка)
4. Скопируй токен
5. При `git push` вместо пароля вставь этот токен

```bash
# Чтобы не вводить каждый раз:
git config --global credential.helper store
```

### Вариант 3: SSH-ключ (для продвинутых)
```bash
# Генерируем ключ
ssh-keygen -t ed25519 -C "твой@email.com"

# Копируем публичный ключ
cat ~/.ssh/id_ed25519.pub

# Вставляем на GitHub: Settings → SSH and GPG Keys → New SSH Key
```

## Полезные команды

| Команда | Что делает |
|---------|-----------|
| `pwd` | Показать текущую папку |
| `ls` (Mac/Linux) / `dir` (Win) | Список файлов |
| `cd название_папки` | Зайти в папку |
| `cd ..` | Выйти на уровень выше |
| `mkdir новая_папка` | Создать папку |
| `touch файл.txt` (Mac) | Создать пустой файл |
| `code .` | Открыть текущую папку в Cursor |

## Если что-то пошло не так

```bash
# Отменить все изменения (вернуть как было)
git checkout .

# Откатить последний коммит (сохранив файлы)
git reset --soft HEAD~1

# Посмотреть историю коммитов
git log --oneline

# Удалить привязку к GitHub (если ошибся)
git remote remove origin
```
