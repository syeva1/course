# Worktree и параллельные агенты в Cursor

> Как запускать несколько AI-агентов одновременно.
> Источник: официальный гайд Cursor + @misha_davai_po_novoi

---

## Что такое Worktree

Worktree — это механизм Git, который позволяет работать с **несколькими ветками одновременно** в разных папках.

Обычно: сохранил → переключился на ветку → поработал → переключился обратно.
С Worktree: каждая ветка в своей папке, работают параллельно.

---

## Зачем это в Cursor

Cursor создаёт **изолированные копии проекта** для каждого агента:
- Агенты не мешают друг другу
- Каждый может менять, собирать и тестировать код
- Твоя основная копия не затрагивается

---

## Как запустить

1. Открой дропдаун агента (справа от поля ввода)
2. Выбери **Worktree**
3. Напиши задачу → агент работает в изолированной копии
4. Когда закончит → нажми **Apply** для слияния изменений

---

## Мощный паттерн: несколько моделей

Запусти **один и тот же промпт через разные модели** одновременно:

1. Выбери несколько моделей из дропдауна
2. Отправь промпт
3. Сравни результаты бок о бок
4. Cursor предложит, какой вариант лучше

### Когда полезно:
- Сложные задачи — разные модели пробуют разные подходы
- Сравнение качества кода
- Поиск краевых случаев, которые одна модель пропустила

---

## Облачные агенты

Для задач, которые можно делегировать «на фон»:

- Багфиксы, которые всплыли во время другой работы
- Рефакторинг
- Генерация тестов
- Обновление документации

### Как работает:
1. Запусти с **cursor.com/agents** (или из редактора, или с телефона)
2. Агент клонирует репозиторий в песочницу
3. Работает автономно
4. Открывает **Pull Request**
5. Ты получаешь уведомление → ревьюишь → мержишь

Можно закрыть ноутбук — агент работает в облаке.

---

## Debug Mode

Когда обычный «исправь баг» не помогает:

1. Генерирует несколько **гипотез** о причине бага
2. Добавляет **логирование** в подозрительные места
3. Просит **воспроизвести** баг
4. Анализирует **реальные данные**
5. Делает **точечное исправление**
6. Убирает добавленные логи

### Когда использовать:
- Баг воспроизводится, но непонятно почему
- Race conditions (порядок выполнения)
- Проблемы производительности
- Регрессии (раньше работало)
