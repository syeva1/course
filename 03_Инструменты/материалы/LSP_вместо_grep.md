# LSP вместо Grep — ускорение навигации

> Как сделать Cursor в 10 раз быстрее при работе с кодом.
> Источник: @misha_davai_po_novoi

---

## Проблема

Когда AI ищет что-то в коде через `grep` — он читает **все файлы** подряд. Это медленно (30-60 секунд) и тратит токены.

## Решение: LSP

LSP (Language Server Protocol) — встроенная навигация по коду в Cursor. Работает за **доли секунды**.

---

## Что умеет LSP

| Функция | Что делает | Аналогия |
|---------|-----------|----------|
| `workspaceSymbol` | Найти, где что-то определено | «Где живёт эта функция?» |
| `findReferences` | Все места использования | «Кто вызывает эту функцию?» |
| `goToDefinition` | Перейти к исходнику | «Покажи код этой функции» |
| `goToImplementation` | Найти реализацию | «Как это работает внутри?» |
| `hover` | Инфо о типе без чтения файла | «Что это за переменная?» |

---

## Правило для .mdc

Добавь в `.cursor/rules/navigation.mdc`:

```
---
description: "Навигация по коду через LSP"
globs: ["**/*"]
alwaysApply: true
---

## Навигация по коду

Для навигации используй LSP вместо Grep и Read:
- workspaceSymbol — найти, где что-то определено
- findReferences — все места использования
- goToDefinition / goToImplementation — перейти к исходнику
- hover — информация о типе без чтения файла

Grep используй только когда LSP недоступен (текстовые файлы, конфиги).
```

---

## Результат

| Без LSP | С LSP |
|---------|-------|
| grep по всем файлам: 30-60 сек | Прямой переход: < 1 сек |
| Читает лишние файлы → тратит токены | Точечный доступ → экономия |
| Может найти не тот файл | Всегда точный результат |
