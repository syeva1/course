# Как работать с агентами в Cursor: полный гайд

> Официальные практики от команды Cursor.
> Перевод и адаптация: @misha_davai_po_novoi

---

## Три режима работы

| Режим | Что делает | Когда использовать |
|-------|-----------|-------------------|
| **Ask** | Отвечает на вопросы, ничего не меняет | Понять, как что-то работает |
| **Plan** | Составляет план, ждёт одобрения | Новые задачи, когда не уверен |
| **Agent** | Делает всё сам: файлы, код, тесты | Когда понимаешь, чего ожидать |

**Начинай с Plan** → привыкнешь → переходи на Agent.

---

## Главное правило: ПЛАНИРУЙ ПЕРЕД КОДОМ

Исследование Чикагского университета: опытные разработчики **всегда** планируют перед генерацией кода.

### Как включить Plan Mode
1. Открой чат (Cmd+L)
2. Нажми **Shift+Tab** → включится Plan Mode
3. Опиши задачу
4. Cursor составит план → ты одобришь → он начнёт работать

### Сохраняй планы
Кнопка «Save to workspace» → план сохранится в `.cursor/plans/`
- Документация для команды
- Можно продолжить прерванную работу
- Контекст для будущих сессий

## Контекст — самое важное

### Не перегружай
Контекстное окно — это «оперативная память» AI. Чем меньше шума — тем точнее результат.

### Пусть Cursor ищет сам
Не нужно вручную тегать каждый файл. У Cursor два типа поиска:
- **Grep** — точные совпадения (слово «auth» → все места где оно есть)
- **Семантический** — понимает смысл («аутентификация» → найдёт loginUser)

### Когда начинать новый чат

**Новый чат:**
- Перешёл к другой задаче
- Cursor повторяет одни и те же ошибки
- Закончил логическую единицу работы

**Продолжай чат:**
- Итерируешь ту же фичу
- Cursor нужен контекст из начала разговора
- Дебажишь то, что он только что сделал

### Ссылки на прошлую работу
В новом чате используй **@Past Chats** — Cursor подтянет контекст из предыдущих разговоров.

## Rules: постоянные правила

Rules = файлы в `.cursor/rules/`, которые Cursor читает **при каждом разговоре**.

### Пример rules (код)
```markdown
# Команды
- `npm run build`: Собрать проект
- `npm run test`: Запустить тесты

# Стиль кода
- Python 3.10+
- Комментарии на русском
- Типы данных (type hints) обязательны

# Воркфлоу
- Всегда проверяй код после изменений
- Ключи только из .env
```

### Когда добавлять rules
**Не заранее!** Добавляй rule **только когда заметил**, что Cursor повторяет одну и ту же ошибку.

## Skills: динамические возможности

Skills = специализированные инструкции, которые Cursor загружает **по необходимости**.

В отличие от Rules (всегда включены), Skills активируются только когда релевантны. Это экономит контекстное окно.

## Параллельные агенты (Worktree)

Cursor позволяет запускать **несколько агентов одновременно**:
1. Выбери «worktree» в дропдауне агента
2. Каждый агент работает в своей изолированной копии
3. Когда закончит → нажми Apply для слияния

### Мощный паттерн: несколько моделей
Запусти один и тот же промпт через **разные модели** одновременно → сравни результаты → выбери лучший.

---

## Облачные агенты

Задачи, которые можно делегировать:
- Багфиксы, которые всплыли во время другой работы
- Рефакторинг
- Генерация тестов
- Обновление документации

Запускай с cursor.com/agents → агент клонирует репо → работает → открывает PR.

---

## Типовые воркфлоу

### Test-Driven Development (TDD)
1. Попроси написать **тесты** (не код!)
2. Проверь, что тесты **падают**
3. **Закоммить тесты отдельно**
4. Попроси написать **код, который проходит тесты**
5. Закоммить код

Зачем отдельно? Чтобы AI не подогнал тесты под свой код.

### Изучение кодовой базы
Задавай вопросы как коллеге:
- «Как работает авторизация?»
- «Как добавить новый API-эндпоинт?»
- «Почему тут вызывается setUser() а не createUser()?»

### Ревью кода
- Во время генерации: следи за диффами, Escape если не туда
- После: Review → Find Issues
- Автоматически: Bugbot на каждый PR

## Работа с картинками

Cursor понимает изображения:
- Вставь макет дизайна → «Реализуй этот интерфейс»
- Скриншот бага → «Почему так выглядит?»
- Фото референса → «Сделай в таком стиле»

---

## Главные принципы успешной работы

1. **Конкретные промпты** > расплывчатые
   - ❌ «добавь тесты для auth.ts»
   - ✅ «Напиши тест для auth.ts: logout + краевые случаи, паттерны из __tests__/»

2. **Итерируй настройки** — начни просто, усложняй по мере надобности

3. **Ревьюй внимательно** — AI-код может выглядеть правильно, но содержать ошибки

4. **Давай проверяемые цели** — тесты, типы, линтеры

5. **Относись как к коллеге** — проси план, требуй объяснений, возражай
