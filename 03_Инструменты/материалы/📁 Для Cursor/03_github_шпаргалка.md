# GitHub — шпаргалка для новичка

## Что есть что

| Термин | Что это |
|--------|---------|
| **Repository (репо)** | Папка проекта на GitHub |
| **Commit** | Сохранение изменений с комментарием |
| **Push** | Отправка коммитов на GitHub |
| **Pull** | Получение изменений с GitHub |
| **Clone** | Скачать проект с GitHub к себе |
| **Branch** | Ветка — параллельная версия проекта |
| **Merge** | Объединение веток |
| **Fork** | Копия чужого проекта в ваш аккаунт |

## Первый раз — настройка

```bash
# Представьтесь Git'у (один раз)
git config --global user.name "Ваше Имя"
git config --global user.email "email@example.com"
```

## Ежедневные команды

```bash
# Создать новый проект
git init
git add .
git commit -m "Первый коммит"
git remote add origin https://github.com/user/repo.git
git push -u origin main

# Обычная работа
git add .                    # Добавить изменения
git commit -m "Что сделал"   # Сохранить
git push                     # Отправить на GitHub
git pull                     # Получить обновления

# Посмотреть состояние
git status                   # Что изменилось?
git log --oneline            # История коммитов
```

## Через Cursor (без терминала)

1. **Source Control** → иконка ветки слева
2. Файлы с изменениями видны в списке
3. Нажмите **+** чтобы добавить (stage)
4. Напишите сообщение коммита
5. Нажмите **✓ Commit**
6. Нажмите **Push** (или Publish Branch)

## Правила хорошего коммита

✅ `Добавил форму обратной связи`
✅ `Исправил баг с отображением цены`
✅ `Обновил стили главной страницы`

❌ `fix`
❌ `update`
❌ `asdfgh`

## Если что-то пошло не так

```bash
# Отменить изменения в файле (до коммита)
git checkout -- <файл>

# Отменить последний коммит (сохранив файлы)
git reset --soft HEAD~1

# Посмотреть, что изменилось
git diff
```
