Tab — автодополнение

Когда использовать: ты уже пишешь и хочешь ускориться

  • Cursor предсказывает, что вы напишете дальше
  • Серый текст — предложение AI. Нравится? Нажмите Tab
  • Не нравится? Просто пишите дальше — предложение исчезнет

Конкретный пример: Ты пишешь HTML-страницу. Набрал <div class="price — и Tab предлагает целый блок с ценой, кнопкой и описанием. Одно нажатие — готово. Без Tab ты бы набирал это 2 минуты.

Когда именно Tab Рутинные повторяющиеся конструкции: формы, списки, похожие блоки

АССИСТ+

Cmd+K — быстрая правка на месте

Когда использовать: нужно изменить конкретный кусок кода

  • Выделите код → нажмите Cmd+K (Win: Ctrl+K)
  • Напишите, что нужно изменить — AI исправит прямо на месте
  • Примите изменения или отклоните

Конкретный пример: У тебя на сайте кнопка «Submit». Выделяешь её код → Cmd+K → «Сделай кнопку красной, округлой, с тенью и текстом "Отправить заявку"». Через 3 секунды — готово, без гугления CSS.

Когда именно Cmd+K Поменять стиль, перевести текст, добавить проверку — точечная правка

АССИСТ+

Cmd+L — AI Chat

Когда использовать: нужно спросить, разобраться, обсудить

  • Откройте чат: Cmd+L (Win: Ctrl+L)
  • AI видит ваш файл и отвечает с учётом контекста
  • Можно прикрепить файлы через @файл

Конкретный пример: Ты скачал чужой проект и не понимаешь, что происходит. Cmd+L → «Объясни простым языком, что делает этот файл и как связан с остальными». AI разложит по полочкам — как будто спросил у коллеги.

Когда именно Cmd+L Понять чужой код, найти баг, спросить «а как лучше сделать?»

АССИСТ+

Cmd+I — Composer (Агент)

Когда использовать: большая задача на несколько файлов

  • Откройте Composer: Cmd+I (Win: Ctrl+I)
  • AI создаёт и редактирует несколько файлов сразу
  • Показывает diff — вы решаете, что принять
  • Главный Самый мощный инструмент Cursor

Конкретный пример: «Создай страницу с формой заявки: HTML-файл, CSS-стили, JS-валидация (проверка телефона и email), и добавь отправку в Telegram-бот». Cmd+I создаст 3–4 файла разом — тебе останется проверить и принять.

Когда именно Cmd+I Новые фичи, новые страницы, масштабные изменения

GitHub — зачем лично тебе

Три реальные ситуации, когда GitHub спасает

  • 🔥 Потерял файлы — ноутбук сломался, случайно удалил папку. С GitHub — зашёл на сайт, скачал всё обратно за 1 минуту
  • ⏪ Сломал проект — AI внёс изменения, всё перестало работать. С GitHub — откатил к вчерашней версии одной командой
  • 👥 Работаешь с кем-то — дизайнер правит стили, ты — логику. Без GitHub — пересылаете zip-файлы. С GitHub — каждый работает в своей копии, потом объединяете
  • 📋 Портфолио — «Покажи, что умеешь» → даёшь ссылку на GitHub

GitHub — создание аккаунта

Займёт 2 минуты

  • 1 Откройте github.com
  • 2 Нажмите Sign up
  • 3 Введите email, придумайте username и пароль
  • 4 Подтвердите email
  • 5 Готово! Бесплатный план — всё, что нужно
АССИСТ+

Подключение GitHub к Cursor

Source Control — встроенный Git-клиент

  • 1 В Cursor откройте Source Control (иконка ветки слева)
  • 2 Нажмите Initialize Repository
  • 3 Cursor предложит войти в GitHub — разрешите
  • 4 Авторизуйтесь в браузере
  • 5 Готово — Cursor связан с GitHub!
АССИСТ+

Первый Push

Сохраняем проект в облако

  • 1 Внесите изменения в файл
  • 2 В Source Control увидите список изменений
  • 3 Напишите commit message — кратко, что сделали
  • 4 Нажмите ✓ Commit
  • 5 Нажмите Publish Branch (или Push)

Совет Коммитьте часто, пишите понятные сообщения

АССИСТ+

GitHub через терминал

Подключаем папку к репозиторию командами

Открой терминал в Cursor: Ctrl+` (внизу экрана)

  • 1 git init — включить Git в папке
  • 2 git add . — добавить все файлы
  • 3 git commit -m "первый коммит" — сохранить
  • 4 git remote add origin https://github.com/ТВОЙ_НИК/РЕПО.git
  • 5 git push -u origin main — отправить на GitHub

ЛАЙФХАК Не помнишь команды? Просто напиши агенту в чат: «Подключи эту папку к моему GitHub-репозиторию» — он сам выполнит все команды в терминале.

АССИСТ+

Pull и совместная работа

Как получить изменения других

  • Pull — скачивает изменения из GitHub на ваш компьютер
  • Push — отправляет ваши изменения в GitHub
  • Всегда делайте Pull перед Push, чтобы не было конфликтов
  • Clone — скачать чужой проект с GitHub к себе
  • Правило Pull → работа → commit → push

LSP — ускорение навигации

Language Server Protocol — ваш невидимый помощник

  • LSP — протокол, который даёт редактору «понимание» кода
  • Подсветка ошибок в реальном времени
  • Go to Definition — перейти к определению функции (Cmd+Click)
  • Find All References — где используется переменная
  • Автодополнение имён функций, переменных, методов
  • Инсайт LSP работает «из коробки» для большинства языков
