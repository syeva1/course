## Все @-символы

| Символ | Что делает | Когда использовать |
|--------|-----------|-------------------|
| `@File` | Подключает конкретный файл | Работа с одним файлом, точечные правки |
| `@Folder` | Подключает всю папку | Работа с модулем/компонентом |
| `@Codebase` | Анализирует весь проект | Глобальные вопросы, рефакторинг, поиск связей |
| `@Web` | Ищет в интернете | Актуальная информация, тренды, документация |
| `@Doc` | Подключает документацию | Работа с API, фреймворками, библиотеками |
| `@Git` | История изменений | Code review, поиск ошибок, changelog |
| `@Notepad` | Блокнот с заметками | Постоянный контекст проекта |

---

## Источники контекста (по приоритету)

```
1. .cursorrules          →  Постоянные правила (AI читает всегда)
2. Notepad               →  Заметки о проекте (подключаешь через @Notepad)
3. @File / @Folder       →  Конкретные файлы (точечный контекст)
4. @Codebase             →  Весь проект (тяжёлый, но мощный)
5. @Doc                  →  Внешняя документация
6. @Web                  →  Интернет (самый свежий, но не всегда точный)
7. @Git                  →  История изменений
```

---

## Формула хорошего запроса

```
@[источник контекста] + короткий запрос = отличный результат
```

**Примеры:**

| Плохо ❌ | Хорошо ✅ |
|---------|----------|
| Напиши текст для сайта | `@Codebase` Напиши текст для главной, учитывая .cursorrules |
| Как сделать адаптивный дизайн | `@Doc Tailwind CSS` Как сделать адаптивную сетку из 3 колонок |
| Что изменилось в проекте | `@Git` Покажи изменения за последнюю неделю |
| Исправь ошибку в стилях | `@File styles.css` Кнопка не центрируется — исправь |

---

## Чего НЕ делать

| Ошибка | Почему плохо |
|--------|-------------|
| Не давать контекст вообще | AI гадает, результат абстрактный |
| Использовать `@Codebase` для мелочей | Медленно и дорого по токенам |
| Писать контекст в каждом промпте | Лучше один раз настроить .cursorrules |
| Игнорировать структуру проекта | AI читает имена файлов — хаос = плохой контекст |
| Забывать про Notepad | Удобный способ хранить контекст без повторений |
