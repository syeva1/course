# Инструкция — Модуль 02. Контекст

## Шаг 1. Пойми, что такое контекст

Контекст — это всё, что AI знает о тебе, твоём проекте и задаче в момент разговора.

Без контекста AI — как стажёр в первый день на работе. Он умный, но ничего не знает о твоей компании, клиентах, продуктах.

Чем больше контекста ты даёшь AI, тем лучше результат. Это главное правило всего курса.

Попробуй прямо сейчас. Открой AI Chat (Cmd+L на Mac, Ctrl+L на Windows) и напиши:

```
Напиши текст для главной страницы
```

Результат будет абстрактный и бесполезный. AI не знает кто ты, что за бизнес, для кого страница. Запомни этот результат — мы к нему вернёмся.

---

## Шаг 2. Символ @ — главный инструмент контекста

В Cursor символ `@` — это способ сказать AI: «Посмотри сюда!»

Открой AI Chat и набери `@` — появится выпадающий список. Вот что там есть:

**@File** — подключает конкретный файл. AI прочитает его целиком.

Как попробовать:
1. В чате набери `@File`
2. Выбери свой .cursorrules (или любой другой файл)
3. Напиши: «Опиши что написано в этом файле»
4. AI ответит, зная содержимое файла

Результат: AI точно описывает содержимое выбранного файла.

**@Folder** — подключает целую папку. AI видит все файлы в ней.

Как попробовать:
1. В чате набери `@Folder`
2. Выбери папку проекта
3. Напиши: «Какие файлы тут есть и для чего они?»

Результат: AI перечисляет файлы и объясняет структуру.

**@Codebase** — подключает весь проект. Самый мощный, но самый дорогой по токенам.

Как попробовать:
1. В чате набери `@Codebase`
2. Напиши: «Опиши структуру этого проекта и предложи улучшения»
3. AI проанализирует все файлы

Когда использовать: когда нужно понять проект целиком, найти связи между файлами, провести рефакторинг.

Когда НЕ использовать: для мелких задач в одном файле — будет медленнее и дороже.

Результат: AI даёт обзор всего проекта, а не одного файла.

## Шаг 3. @Web — свежая информация из интернета

AI обучен на данных до определённой даты. Если нужна свежая информация — используй @Web.

1. В AI Chat набери `@Web`
2. Напиши: «Какие AI-инструменты для бизнеса появились в 2026 году?»
3. AI найдёт и обобщит свежую информацию из интернета

Когда использовать: актуальные цены, новые инструменты, свежие тренды, проверка фактов.

Результат: AI отвечает со ссылками на источники, информация свежая.

---

## Шаг 4. @Doc — подключение документации

@Doc подключает документацию внешних сервисов и библиотек.

1. В AI Chat набери `@Doc`
2. Из списка выбери нужную документацию
3. Если нужной нет — добавь свою:
   - Зайди в Cursor → Settings (шестерёнка внизу слева) → Features → Docs
   - Нажми «Add new doc»
   - Вставь URL документации (например, https://core.telegram.org/bots/api)
   - Cursor проиндексирует её

Теперь можешь писать:
```
@Doc Telegram Bot API Как отправить фото через бота?
```

AI ответит на основе актуальной документации, а не из своей памяти.

Результат: AI даёт точные ответы по API, не выдумывая несуществующие методы.

## Шаг 5. @Git — история изменений

@Git даёт AI доступ к истории изменений проекта (если ты используешь Git).

1. В чате набери `@Git`
2. Спроси: «Какие изменения были за последнюю неделю?»
3. AI покажет коммиты и что менялось

Когда полезно: разобраться что менялось, найти причину ошибки, составить changelog.

Если ты ещё не используешь Git — не страшно, мы подключим его в Модуле 03.

Результат: AI показывает историю изменений в понятном виде.

## Шаг 6. Notepad — постоянная заметка для AI

Notepad — блокнот внутри Cursor, который AI видит при каждом запросе.

Как открыть:
1. Нажми Cmd+Shift+P (Mac) или Ctrl+Shift+P (Windows)
2. Набери «Notepad» и выбери «Notepad: Open»
3. Создай заметку

Что туда написать — пример:

```
Проект: сайт агентства по подбору бизнес-ассистентов
Аудитория: предприниматели с выручкой от 1 млн/мес
Стек: HTML, CSS, JavaScript
Тон: профессиональный, но дружелюбный
Не использовать: jQuery, Bootstrap
Все тексты: на русском языке
```

Теперь в чате используй `@Notepad` — AI учтёт эту информацию.

Разница между Notepad и .cursorrules:
- .cursorrules — AI читает ВСЕГДА, автоматически
- Notepad — AI читает только когда ты пишешь @Notepad

Результат: AI учитывает информацию из Notepad при ответе.

## Шаг 7. Структура проекта = контекст

AI читает имена файлов и папок. Понятные имена = лучший контекст = лучший результат.

Плохо:
```
project/
├── file1.html
├── file2.css
├── stuff.js
└── old_copy_final_v2.html
```

Хорошо:
```
project/
├── index.html
├── css/
│   └── styles.css
├── js/
│   └── main.js
├── assets/
│   └── logo.png
└── .cursorrules
```

Что делать прямо сейчас:
1. Открой свой проект из Модуля 01
2. Переименуй файлы, если они называются непонятно
3. Создай папки для разных типов файлов (css, js, assets)
4. Удали дубликаты и мусор

Результат: Когда ты потом используешь @Codebase, AI сразу поймёт что где лежит.

## Шаг 8. Файл О_СЕБЕ.md — расскажи AI о себе

Это одна из самых сильных техник. Создай файл, в котором AI узнает тебя как человека.

Зачем: AI подстроит ответы под твой уровень, стиль, задачи. Вместо абстрактных советов — конкретные рекомендации для тебя.

1. Создай файл `О_СЕБЕ.md` в корне проекта
2. Заполни по шаблону из `материалы/шаблон_файла_о_себе.md`
3. Включи:
   - Как тебя зовут и чем занимаешься
   - Твои сильные стороны
   - Как ты общаешься (формально / просто / с юмором)
   - Какие цели прямо сейчас
   - Какие инструменты используешь
   - С кем работаешь

Проверка — в AI Chat напиши:
```
@File О_СЕБЕ.md Дай мне 3 совета по развитию моего бизнеса
```

Результат: AI даёт советы, учитывая твою конкретную ситуацию, а не абстрактные рекомендации.

## Шаг 9. Создай правило для автозагрузки личного контекста

Чтобы не писать @File О_СЕБЕ.md каждый раз, можно создать .mdc правило.

1. Создай папку `.cursor/rules/` в проекте (если её нет)
2. Создай файл `.cursor/rules/personal.mdc`
3. Содержимое:

```
---
description: Личный контекст пользователя
globs: *
alwaysApply: true
---

При ответе учитывай информацию из файла О_СЕБЕ.md в корне проекта.
Подстраивай стиль, примеры и рекомендации под этого конкретного человека.
```

Теперь AI будет учитывать твой личный контекст при каждом запросе, автоматически.

Результат: Каждый ответ AI персонализирован без дополнительных усилий.

## Шаг 10. Система папок для жизни

Cursor — не только для кода. Ты можешь создать систему папок для организации своей жизни, и AI поможет с ней работать.

Подробная инструкция — в `материалы/структура_папок_жизнь.md`. Коротко:

1. Создай папку «Моя_жизнь» и открой её как проект в Cursor
2. Внутри создай разделы: Документы, Здоровье, Финансы, Обучение, Цели
3. Создай папку «Распределение» — это входящие. Кидай туда всё подряд, AI разберёт.
4. Добавь README.md с описанием структуры
5. Добавь .cursorrules с правилами работы

Потом пиши AI:
```
Разбери все файлы из папки Распределение — разложи по нужным разделам
```

Результат: У тебя организованная система, где AI — твой секретарь.

## Главное правило этого модуля

Базы знаний > промпты.

Длинный промпт забывается после одного запроса. Хороший контекст (.cursorrules, Notepad, О_СЕБЕ.md, структура проекта) работает всегда, при каждом запросе, без усилий.

Не пиши длинные промпты. Настрой контекст один раз — и AI будет давать отличные результаты при коротких запросах.
