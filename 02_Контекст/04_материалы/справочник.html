TDD в Cursor

Зачем TDD

«Напиши функцию» → AI может написать что угодно. «Напиши тест, потом код» → тест становится контрактом. AI вынужден написать код, который проходит тест.

Пошаговый процесс

  • 1 Напиши тесты (не код!) — обычный случай, границы, ошибки
  • 2 Убедись, что тесты падают
  • 3 Закоммить тесты отдельно
  • 4 Напиши реализацию, не меняя тесты
  • 5 Закоммить код

Промпт для TDD

Я хочу функцию [ОПИСАНИЕ]. Этап 1: Напиши тесты (обычный случай, границы, ошибки). НЕ пиши реализацию. Этап 2 (после одобрения): Напиши реализацию. НЕ меняй тесты.

Как создать

Папка .cursor/rules/, файл style.mdc:

--- description: Правила стилей для React globs: **/*.tsx alwaysApply: false --- # Правила - Tailwind CSS - Компоненты в отдельных файлах

Поля заголовка

ПолеНазначение
descriptionКогда применять правило
globs**/*.py, src/api/**, !**/*.test.ts
alwaysApplytrue = всегда, false = только при совпадении globs

Создать за 1 минуту

  • 1 Скопируй структуру из материала
  • 2 Cursor → Cmd+I (Composer)
  • 3 Вставь и допиши промпт (ниже)
  • 4 Accept → готово
Создай эту структуру папок в текущей директории. Для каждой папки создай README.md с описанием: что хранится, когда обращаться, примеры файлов.

Зачем README в каждой папке

README — навигация для AI. Cursor понимает: что лежит в папке, зачем она, какие файлы ожидать.

Без README → AI не понимает контекст. С README → AI знает, где что искать.
