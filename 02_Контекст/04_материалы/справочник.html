Структура папок и правила .mdc

Модульные правила вместо одного файла

  • Структура «Моя жизнь» — README.md, О_СЕБЕ.md, папки по темам (работа, здоровье, финансы)
  • .mdc формат — заменил устаревший .cursorrules
  • Один файл = одна тема — .cursor/rules/style.mdc, tone.mdc, stack.mdc
  • Можно включать по условию — правило активируется только когда нужно
  • Важно Называй файлы понятно: не file1.html, а index.html

Базы знаний вместо промптов

Главный инсайт 2026

  • Промпт — пишешь инструкцию каждый раз заново. Результат нестабильный
  • База знаний — один раз создал файлы с примерами, тоном, правилами. AI читает их всегда
  • docs/ — папка с описанием проекта, стиля, аудитории
  • Примеры решают — 2-3 хороших примера важнее длинной инструкции
  • Совет Сохраняй лучшие результаты AI в папку examples/

Примеры запросов и TDD

С контекстом vs без контекста

  • Без контекста: «Напиши текст для сайта» — шаблонный результат
  • С контекстом: AI знает бизнес, аудиторию, тон — результат персонализирован
  • TDD — Test-Driven Development: сначала тест, потом код
  • Зачем TDD? — тест становится контрактом. AI обязан написать код, который проходит тест
  • Совет Попроси: «Сначала напиши тест, потом реализацию»

FAQ

Ответы на частые вопросы

  • Сколько информации давать? — Чем больше, тем лучше. AI отфильтрует лишнее сам
  • .cursorrules ещё работает? — Пока да, но он deprecated. Переходи на .mdc
  • Можно ли использовать Notepad? — Да, это быстрый способ добавить контекст в сессию
  • AI забывает контекст между чатами? — Да. Поэтому важны файлы в проекте, а не устные инструкции
  • Как обновлять контекст? — Регулярно обновляй docs/ и .mdc файлы по мере развития проекта
  • TDD обязательно? — Нет, но для сложных задач он значительно улучшает результат
