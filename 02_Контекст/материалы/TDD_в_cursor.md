# TDD в Cursor: сначала тесты, потом код

> Test-Driven Development — лучший способ получить рабочий код от AI.
> Источник: официальный гайд Cursor

---

## Зачем это нужно

Когда просишь AI «напиши функцию» — он может написать что угодно. Проверить сложно.

Когда просишь AI «напиши тест, потом код» — тест становится **контрактом**. AI вынужден написать код, который проходит тест.

---

## Пошаговый процесс

### Шаг 1: Напиши тесты (не код!)

```
Напиши тесты для функции calculate_price():
- Обычная цена: 1000 * 1 = 1000
- Скидка 10%: 1000 * 0.9 = 900
- Нулевое количество: ошибка ValueError
- Отрицательная цена: ошибка ValueError

Это TDD — не пиши реализацию, только тесты.
```

### Шаг 2: Убедись, что тесты падают

```
Запусти тесты и убедись, что они падают (функции ещё нет).
НЕ пиши код реализации на этом этапе.
```

### Шаг 3: Закоммить тесты

```bash
git add tests/
git commit -m "test: тесты для calculate_price"
```

**Зачем отдельный коммит?**
- Фиксирует «контракт» — что именно должен делать код
- Если AI потом что-то сломает → откатываешься к тестам
- AI не сможет подогнать тесты под свой код

### Шаг 4: Напиши код

```
Напиши реализацию calculate_price(), которая проходит все тесты.
НЕ меняй тесты. Итерируйся, пока все не пройдут.
```

### Шаг 5: Закоммить код

```bash
git add src/
git commit -m "feat: реализация calculate_price"
```

---

## Почему это лучше обычного подхода

| Обычный | TDD |
|---------|-----|
| «Напиши функцию» → непонятно, правильно ли | Тесты доказывают, что работает |
| AI может подогнать тесты под код | Тесты зафиксированы отдельным коммитом |
| Сломал → непонятно что | Сломал → откатился к тестам → попробовал снова |
| Проверяешь вручную | Автоматическая проверка |

---

## Промпт для быстрого TDD

```
Я хочу функцию, которая [ОПИСАНИЕ].

Этап 1: Напиши тесты. Покрой:
- Обычный случай
- Граничные случаи (пустой ввод, нулевые значения)
- Ошибки (неправильный тип, отрицательные числа)
НЕ пиши реализацию.

Этап 2 (после моего одобрения тестов):
Напиши реализацию. НЕ меняй тесты.
Итерируйся, пока все тесты не пройдут.
```
