# ✅ Чеклист: Правильно ли я даю контекст AI?

> Пройди этот чеклист перед каждым серьёзным запросом к AI

---

## Перед началом работы

- [ ] `.cursorrules` создан и заполнен
- [ ] В `.cursorrules` указано: кто ты, что за проект, какой стек, какой стиль
- [ ] Файлы и папки названы понятно (не `file1.html`, а `index.html`)
- [ ] Структура проекта логичная (css/, js/, assets/ и т.д.)
- [ ] Notepad настроен с ключевой информацией о проекте

---

## При каждом запросе

- [ ] Использую `@` для подключения нужного контекста
- [ ] Для точечных задач — `@File` (не `@Codebase`)
- [ ] Для глобальных вопросов — `@Codebase`
- [ ] Для актуальной информации — `@Web`
- [ ] Запрос короткий и конкретный (не «сделай красиво»)

---

## Если результат плохой

- [ ] Проверь: дал ли ты AI контекст?
- [ ] Попробуй другой @-символ
- [ ] Добавь уточнение в Notepad
- [ ] Обнови `.cursorrules`, если проект изменился
- [ ] Разбей задачу на части — AI лучше работает с конкретными запросами

---

## Красные флаги 🚩

| Признак | Проблема |
|---------|----------|
| AI выдаёт абстрактный текст | Нет контекста о проекте |
| AI использует неправильный стек | Стек не указан в .cursorrules |
| AI не знает про твой бизнес | Нет описания в .cursorrules / Notepad |
| AI даёт устаревшую информацию | Используй `@Web` для свежих данных |
| AI путает файлы | Имена файлов неинформативные |
