# Базы знаний > Промпты

> Главный инсайт 2026: не пиши промпты — создавай базы знаний.
> Источник: @misha_davai_po_novoi (инсайт Миши Попова)

---

## Идея

**Старый подход (промпты):**
```
Напиши пост для Telegram. Тон: дружелюбный. Длина: 500 символов.
Используй эмодзи. Начни с вопроса. Заверши CTA.
```
Каждый раз пишешь заново. Каждый раз результат разный.

**Новый подход (базы знаний):**
```
Создай папку config/ с файлами:
- style-guide.md  (как писать тексты)
- examples/       (примеры хороших текстов)
- tone.md         (тон и голос бренда)
- rules.md        (что нельзя делать)
```
AI читает всё → сам решает, как применить → результат стабильный.

## Почему это работает лучше

| Промпты | Базы знаний |
|---------|-------------|
| Каждый раз пишешь заново | Написал один раз — используешь всегда |
| Результат непредсказуемый | Результат стабильный |
| Забыл что-то указать → плохой результат | AI сам находит нужное |
| Для одной задачи | Для любых задач в проекте |
| Линейное: промпт → ответ | Контекстное: AI понимает всю картину |

## Как создать базу знаний в Cursor

### 1. Стиль текстов (config/style-guide.md)

```markdown
# Стиль АССИСТ+

## Тон
- Дружелюбный, но профессиональный
- Без канцеляризмов и воды
- Объясняем сложное просто
- Используем «вы» (не «Вы»)

## Структура текста
- Заголовок: вопрос или польза
- Первый абзац: зацепка
- Тело: конкретика, примеры, цифры
- Финал: CTA (призыв к действию)

## Запрещено
- Слова: «уникальный», «инновационный», «синергия»
- Кликбейт и ложные обещания
- Обещания без доказательств
```

### 2. Примеры (config/examples/)

Положи 3-5 текстов, которые тебе нравятся. Cursor прочитает и будет писать в таком же стиле.

### 3. Правила для кода (config/code-rules.md)

```markdown
# Правила кода

- Python 3.10+
- Комментарии на русском
- Docstrings для каждой функции
- Типы данных (type hints)
- Ключи только из .env
- Логирование через logging (не print)
```

### 4. .mdc правило, которое всё связывает

```
---
description: "Читай базу знаний перед любой задачей"
globs: ["**/*"]
alwaysApply: true
---

Перед выполнением любой задачи прочитай:
- config/style-guide.md — стиль текстов
- config/code-rules.md — правила кода
- config/examples/ — примеры хорошего результата
- tasks/lessons.md — уроки из ошибок

Следуй этим правилам автоматически.
```

## Применение

- Контент: style-guide + примеры → AI пишет в твоём стиле
- Код: code-rules + architecture → AI пишет по стандартам
- Продажи: tone + scripts + FAQ → AI отвечает как менеджер
- Любая задача: правила + примеры > один промпт
