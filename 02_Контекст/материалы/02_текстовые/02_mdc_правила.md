# .mdc правила — новый формат Cursor Rules

## Что изменилось?

Файл `.cursorrules` **устарел** (deprecated с Cursor 2.3). Новый формат — файлы `.mdc` в папке `.cursor/rules/`.

## Почему это лучше?

| Старое | Новое |
|--------|-------|
| Один файл `.cursorrules` на весь проект | Много файлов `.mdc` — по одному на тему |
| Всегда подгружается полностью | Можно включать только когда нужно |
| Трудно поддерживать при росте | Модульная структура |

## Как создать?

1. Создай папку `.cursor/rules/` в корне проекта
2. Создай файл с расширением `.mdc`, например `style.mdc`
3. Формат файла:

```
---
description: Правила стилей для React компонентов
globs: **/*.tsx
alwaysApply: false
---

# Правила стилей

- Используй Tailwind CSS
- Компоненты в отдельных файлах
- Нет default export
```

## Поля заголовка

| Поле | Что делает |
|------|-----------|
| `description` | Описание — когда AI должен применять это правило |
| `globs` | Паттерн файлов: `**/*.py`, `src/api/**`, `!**/*.test.ts` |
| `alwaysApply` | `true` = применяется всегда, `false` = только когда совпадает globs |

## Примеры организации

```
.cursor/rules/
├── general.mdc          # Общие правила (alwaysApply: true)
├── react.mdc            # Для React файлов (globs: **/*.tsx)
├── api.mdc              # Для API (globs: src/api/**)
├── testing.mdc          # Для тестов (globs: **/*.test.*)
└── python.mdc           # Для Python (globs: **/*.py)
```

## Совет

Начни с одного файла `general.mdc` с `alwaysApply: true` — это аналог старого `.cursorrules`. Потом добавляй специализированные правила по мере необходимости.
