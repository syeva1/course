# 📖 Инструкция: Контекст — Главный секрет AI

> **Модуль 02** | Курс АССИСТ+

---

## Шаг 1. Пойми, что такое контекст

Контекст — это **всё, что AI знает** о тебе, твоём проекте и задаче в момент разговора.

Без контекста AI — как стажёр в первый день на работе. Он умный, но ничего не знает о твоей компании, клиентах, продуктах. Ты же не будешь злиться на нового сотрудника за то, что он не знает внутренних правил?

**Ключевой принцип:** Чем больше контекста ты даёшь AI, тем лучше результат.

---

## Шаг 2. Изучи символ @

В Cursor символ `@` — это способ сказать AI: «Посмотри сюда!»

1. Открой AI Chat (`Cmd+L` / `Ctrl+L`)
2. Набери `@` — появится выпадающий список доступных источников контекста
3. Попробуй набрать `@File` и выбери любой файл из проекта
4. Задай вопрос про этот файл — AI ответит, зная его содержимое

**Совет:** Привыкай начинать каждый запрос с `@` — это один из главных навыков работы с Cursor.

---

## Шаг 3. Попробуй @Codebase

`@Codebase` — самый мощный инструмент. Он даёт AI доступ ко **всему проекту**.

1. В AI Chat набери: `@Codebase`
2. Задай вопрос: «Опиши структуру этого проекта»
3. AI проанализирует все файлы и даст ответ

**Когда использовать:** Когда нужно понять проект целиком, найти связи между файлами, провести рефакторинг.

**Когда НЕ использовать:** Для мелких задач в одном файле — будет медленнее и дороже по токенам.

---

## Шаг 4. Попробуй @Web

`@Web` позволяет AI искать информацию в интернете прямо из чата.

1. В AI Chat набери: `@Web`
2. Задай вопрос: «Какие актуальные тренды в подборе персонала в 2025 году?»
3. AI найдёт и обобщит свежую информацию

**Когда использовать:** Когда нужна актуальная информация, которой нет в обучающих данных AI.

---

## Шаг 5. Подключи @Doc

`@Doc` подключает документацию внешних сервисов.

1. В AI Chat набери `@Doc`
2. Из списка выбери нужную документацию (или добавь свою)
3. Теперь AI знает API и особенности инструмента

**Как добавить свою документацию:**
- Зайди в Cursor → Settings → Features → Docs
- Нажми «Add new doc»
- Вставь URL документации

---

## Шаг 6. Используй @Git

`@Git` даёт AI доступ к истории изменений проекта.

1. В AI Chat набери `@Git`
2. Спроси: «Какие изменения были сделаны за последнюю неделю?»
3. AI покажет историю коммитов и изменений

**Когда полезно:** Для code review, поиска причин ошибок, составления changelog.

---

## Шаг 7. Настрой Notepad

Notepad — это блокнот внутри Cursor, который AI всегда видит.

1. Открой Notepad: `Cmd+Shift+P` → набери «Notepad»
2. Создай заметку, например:
   ```
   ## О проекте
   Это сайт агентства по подбору бизнес-ассистентов.
   Целевая аудитория: предприниматели с выручкой от 1 млн/мес.
   Стек: HTML, CSS, JavaScript.
   Тон: профессиональный, но дружелюбный.
   ```
3. Теперь при запросах используй `@Notepad` — AI будет учитывать эту информацию

---

## Шаг 8. Продумай структуру проекта

Хорошая структура проекта = хороший контекст для AI.

**Плохо:**
```
project/
├── file1.html
├── file2.css
├── stuff.js
└── old_copy_final_v2.html
```

**Хорошо:**
```
project/
├── index.html
├── css/
│   └── styles.css
├── js/
│   └── main.js
├── assets/
│   └── logo.png
└── .cursorrules
```

AI читает имена файлов и папок. Понятные имена = лучший контекст = лучший результат.

---

## Шаг 9. Вспомни про .cursorrules

В Модуле 01 мы создали файл `.cursorrules`. Теперь ты понимаешь, почему он важен:

**`.cursorrules` — это постоянный контекст.** AI читает его при каждом запросе. Это как инструктаж для стажёра, который висит у него на стене.

Проверь свой `.cursorrules`:
- [ ] Есть описание проекта
- [ ] Указана целевая аудитория
- [ ] Описан стиль и тон
- [ ] Прописаны правила и ограничения

---

## Шаг 10. Запомни главное правило

> **Базы знаний > промпты**

Длинный промпт забывается. Хороший контекст (`.cursorrules`, Notepad, структура проекта, @-символы) — работает всегда.

Не пиши длинные промпты. Настрой контекст один раз — и AI будет давать отличные результаты при коротких запросах.
