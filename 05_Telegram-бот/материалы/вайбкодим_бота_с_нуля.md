# Как создать Telegram-бота в Cursor (пошаговый гайд)

> Полный путь от пустой папки до работающего бота на сервере.
> Источник: вебинар Андрея Погорелого (Нейроцех)

---

## Шаг 1: Регистрация бота

1. Открой Telegram → найди **@BotFather**
2. Отправь `/newbot`
3. Придумай название и username
4. BotFather пришлёт **токен** — скопируй и сохрани

---

## Шаг 2: Создание в Cursor

1. Создай папку для проекта (например, `my-bot/`)
2. Открой в Cursor: **File → Open Folder**
3. Открой чат → режим **Agent**
4. Опиши бота:

```
Напиши Telegram-бота на Python, который будет [ОПИСАНИЕ ЗАДАЧИ].
Вот токен бота: [ВСТАВЬ ТОКЕН]
```

### Примеры описаний:
- «...отвечает на вопросы клиентов по прайсу из файла prices.txt»
- «...модерирует групповой чат, мьютит за ругательства на 1 минуту»
- «...собирает заявки и отправляет их в Google Sheets»
- «...принимает фото и сохраняет в папку по дате»

5. Cursor создаст файлы с кодом
6. Он скажет, какую команду ввести в терминал — **скопируй и нажми Enter**

---

## Шаг 3: Итерации

Первая версия **никогда** не идеальна. Это нормально.

```
Бот не пишет в чате, за что замьютил. Доработай — пусть объясняет причину.
```

```
Добавь команду /help со списком всех функций бота.
```

```
Бот падает, если отправить ему стикер. Исправь.
```

**Цикл:** получил → проверил → описал проблему → Cursor исправил → проверил снова.

## Шаг 4: Деплой на сервер

Бот на твоём компьютере работает только пока Cursor открыт. Для 24/7 нужен сервер.

### 4.1 Арендуй VPS
- **Timeweb** или **Beget** — от 300₽/мес
- Ubuntu 24.04, минимальный тариф

### 4.2 Попроси Cursor составить план

```
У меня есть бот на Python. Хочу задеплоить его на сервер Ubuntu.
Напиши пошаговый план.
```

### 4.3 Подключись к серверу

В терминале Cursor:
```bash
ssh root@[IP_АДРЕС_СЕРВЕРА]
```
Введи пароль из письма хостинга.

### 4.4 Перенеси файлы

```bash
nano bot.py
```
→ Скопируй код из Cursor → Вставь → **Ctrl+O** (сохранить) → **Enter** → **Ctrl+X** (выйти)

### 4.5 Установи зависимости и запусти

```bash
pip install python-telegram-bot python-dotenv
python bot.py
```

## Шаг 5: Фоновая работа (systemd)

Без systemd бот остановится при закрытии SSH.

Скажи Cursor:
```
Сделай так, чтобы бот работал в фоне и запускался автоматически
после перезагрузки сервера. Используй systemd.
```

Cursor создаст конфигурацию и выполнит команды.

### Что systemd делает:
- Бот работает в фоне (можно закрыть терминал)
- Автостарт после перезагрузки сервера
- Автоперезапуск при сбоях

### Полезные команды:
```bash
sudo systemctl status mybot      # Статус бота
sudo systemctl restart mybot     # Перезапустить
sudo journalctl -u mybot -f      # Логи в реальном времени
```
