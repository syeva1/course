# Инструкция — Модуль 05. Telegram-бот для бизнеса

## Шаг 1. Зарегистрируй бота в BotFather

BotFather — это официальный бот Telegram для создания ботов.

1. Открой Telegram
2. В поиске найди `@BotFather`
3. Нажми Start
4. Отправь команду `/newbot`
5. BotFather спросит имя — напиши любое (например: «АССИСТ помощник»)
6. BotFather спросит username — придумай уникальный, обязательно заканчивается на `_bot` (например: `assist_helper_bot`)
7. BotFather выдаст токен — длинная строка вида `123456789:ABCdefGHI_jklMNO-pqrSTUvwxyz`

Этот токен — пароль от бота. Никому не показывай и не публикуй в открытых репозиториях.

Проверка: ты можешь найти своего бота по username в поиске Telegram. Он пока ничего не делает — это нормально.

Результат: Бот создан, токен скопирован.

## Шаг 2. Создай проект в Cursor

1. Создай папку `telegram-bot` (через File Explorer или терминал)
2. Открой её в Cursor: File → Open Folder
3. Создай файлы (правый клик в Explorer → New File):
   - `bot.py` — главный файл бота
   - `config.py` — настройки
   - `.env` — секреты (токен, ключи)
   - `requirements.txt` — зависимости
   - `.gitignore` — что не отправлять на GitHub

4. Заполни `.gitignore`:
```
.env
__pycache__/
*.pyc
Распределение/
```

Зачем .gitignore: файл `.env` содержит секреты. Если его отправить на GitHub — любой сможет управлять твоим ботом.

5. Заполни `requirements.txt`:
```
python-telegram-bot==21.7
aiohttp>=3.9.0
python-dotenv>=1.0.0
```

Результат: Проект создан с правильной структурой файлов.

## Шаг 3. Настрой секреты (.env)

Открой файл `.env` и заполни:

```
TELEGRAM_BOT_TOKEN=вставь_свой_токен_от_BotFather
ADMIN_ID=твой_telegram_id
```

Как узнать свой Telegram ID:
1. Найди бота @userinfobot в Telegram
2. Нажми Start
3. Он покажет твой ID (число вроде 123456789)

Зачем ADMIN_ID: чтобы бот знал, кто хозяин. Некоторые команды (например, статистика) доступны только админу.

Результат: Файл .env заполнен, секреты на месте.

---

## Шаг 4. Напиши config.py

Этот файл загружает секреты из .env и задаёт настройки:

```python
from dotenv import load_dotenv
import os

load_dotenv()

TELEGRAM_BOT_TOKEN = os.getenv("TELEGRAM_BOT_TOKEN")
ADMIN_ID = int(os.getenv("ADMIN_ID", "0"))
DISTRIBUTION_FOLDER = "./Распределение"
```

Зачем load_dotenv: он читает файл `.env` и делает его содержимое доступным через `os.getenv()`. Так секреты не попадают в код.

Результат: Настройки вынесены в отдельный файл.

## Шаг 5. Установи зависимости

Открой терминал в Cursor (Cmd+` или Ctrl+`):

```bash
pip install -r requirements.txt
```

Если Python не установлен:
- Windows: скачай с python.org, при установке поставь галочку «Add to PATH»
- Mac: `brew install python` или скачай с python.org
- Проверка: `python --version` или `python3 --version`

Частая ошибка: `pip: command not found`. Попробуй `pip3 install -r requirements.txt`.

Результат: Библиотеки установлены без ошибок.

## Шаг 6. Напиши минимального бота

Открой `bot.py` и попроси AI написать бота. В Composer (Cmd+I) напиши:

```
Создай простого Telegram-бота на python-telegram-bot 21.7.
Бот должен:
1. Отвечать на /start приветствием
2. Отвечать на /help списком команд
3. Эхо — повторять любое текстовое сообщение

Токен бери из config.py (TELEGRAM_BOT_TOKEN).
Используй ApplicationBuilder.
```

AI создаст код. Нажми Apply.

Проверка — запусти бота:
```bash
python bot.py
```

Открой своего бота в Telegram, нажми Start, напиши что-нибудь. Бот должен повторить.

Чтобы остановить: Ctrl+C в терминале.

Частая ошибка: `Conflict: terminated by other getUpdates request`. Значит уже запущена другая копия бота. Закрой все терминалы и запусти заново.

Результат: Бот запускается, отвечает на /start и эхо.

## Шаг 7. Добавь сохранение контента в папки

Теперь бот будет сохранять всё что ему присылают в папки.

В Composer попроси:

```
Добавь в бота функционал:
1. При запуске создай папку Распределение с подпапками: сообщения, документы, изображения, ссылки
2. Текстовые сообщения → сохраняй в Распределение/сообщения/ как .txt (имя файла: дата_время.txt)
3. Документы → скачивай в Распределение/документы/ с оригинальным именем
4. Фото → скачивай в Распределение/изображения/ как .jpg
5. Если сообщение содержит ссылку (http) → сохраняй ссылку + контекст в Распределение/ссылки/
6. После каждого сохранения отвечай: "Сохранено в [папка]"
```

Проверка:
1. Отправь боту текст → проверь папку `Распределение/сообщения/`
2. Отправь документ → проверь `Распределение/документы/`
3. Отправь фото → проверь `Распределение/изображения/`
4. Отправь ссылку → проверь `Распределение/ссылки/`

Результат: Бот сохраняет контент по папкам. Ты видишь файлы в Explorer.

## Шаг 8. Добавь транскрибацию голосовых

Бот должен превращать голосовые сообщения в текст.

1. Зарегистрируйся на assemblyai.com
2. Скопируй API Key (бесплатно первые 5 часов аудио)
3. Добавь в `.env`: `ASSEMBLYAI_KEY=твой_ключ`
4. В `config.py` добавь: `ASSEMBLYAI_KEY = os.getenv("ASSEMBLYAI_KEY")`
5. Установи: `pip install assemblyai`

В Composer попроси:

```
Добавь обработку голосовых сообщений:
1. Скачай голосовое в temp файл
2. Отправь в AssemblyAI для транскрибации
3. Сохрани транскрипт в Распределение/транскрипты/дата_время.txt
4. Отправь пользователю: "Транскрипт: [текст]. Сохранено."
```

Проверка: отправь голосовое — бот должен ответить текстом транскрипта.

Результат: Голосовые сообщения автоматически превращаются в текст.

В Composer:

```
Добавь команды:
- /status — показать количество файлов в каждой папке Распределение

```

Проверка: отправь /status — бот покажет сколько файлов в каждой папке.

## Шаг 10. Добавь AI-консультанта (опционально)

Если хочешь чтобы бот не просто сохранял, а ещё и отвечал умно:

1. Получи API ключ OpenAI (platform.openai.com) или Claude (console.anthropic.com)
2. Добавь ключ в `.env`
3. В Composer:

```
Добавь режим AI-консультанта:
- По умолчанию бот сохраняет контент
- Если сообщение начинается с "?" — бот отвечает через AI (OpenAI или Claude)
- Системный промпт: информация о бизнесе из .cursorrules
- Добавь кнопки InlineKeyboard: Услуги, Цены, Связаться
```

Результат: Бот умеет и сохранять контент, и отвечать на вопросы через AI.

## Шаг 12. Деплой на сервер

Для постоянной работы бота нужен сервер (VPS).

Варианты хостинга:
- VDSina — от 300 руб/мес, рублёвая оплата
- Timeweb — от 200 руб/мес
- Selectel — от 400 руб/мес

Процесс деплоя:
1. Купи VPS (Ubuntu, минимальный тариф)
2. Подключись по SSH: `ssh root@IP_сервера`
3. Установи Python: `apt update && apt install python3 python3-pip -y`
4. Скопируй файлы на сервер (через Git или scp)
5. Установи зависимости: `pip3 install -r requirements.txt`
6. Создай `.env` на сервере с токеном и ключами
7. Запусти:
```bash
nohup python3 bot.py > bot.log 2>&1 &
```

`nohup` — бот продолжит работать после отключения SSH.

Проверка: закрой терминал, напиши боту в Telegram — должен ответить.

Подробный гайд по деплою — в `материалы/вайбкодим_бота_с_нуля.md`.

Результат: Бот работает на сервере 24/7.
