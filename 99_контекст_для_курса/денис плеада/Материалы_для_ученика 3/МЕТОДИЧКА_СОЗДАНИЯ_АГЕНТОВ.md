## ДЛЯ КОГО ЭТА МЕТОДИЧКА

Ты научишься создавать AI-агентов (AI-сотрудников), которые:
- Выполняют рутинные задачи
- Работают по твоим правилам
- Улучшаются с каждым использованием
- Экономят твоё время

**Важно:** AI-агент — это не волшебная кнопка. Это "супер-стажёр Паша" с IQ 150, которому нужен контекст, чёткие задачи и обратная связь.

---

## 7 ШАГОВ СОЗДАНИЯ АГЕНТА

```
1. КОНТЕКСТ     → Понять, для чего агент
2. ДОЛЖНОСТЬ    → Определить роль и задачи
3. КЕЙСЫ        → Найти лучшие практики
4. АРХИТЕКТУРА  → Спроектировать взаимодействия
5. ПЛАН         → Сформировать и подтвердить
6. СОЗДАНИЕ     → Написать правила и папки
7. ТЕСТИРОВАНИЕ → Проверить и улучшить
```

## ШАГ 1: КОНТЕКСТ

**Цель:** Понять, для какого бизнеса/проекта создаётся агент.

**Что нужно знать:**
- Название компании/проекта
- Чем занимается
- Как общается с клиентами (tone of voice)
- Целевая аудитория

**Где взять:**
- Если есть ВТОРОЙ_МОЗГ.md — прочитай его
- Если нет — спроси у себя или подготовь

**Пример:**
> Компания: Онлайн-школа английского  
> Аудитория: Взрослые 25-45 лет  
> Тон: Дружелюбный, мотивирующий

## ШАГ 3: КЕЙСЫ

**Цель:** Найти лучшие практики для этой должности.

**Где искать:**
1. Google: "[должность] лучшие практики 2024"
2. Google: "[должность] AI assistant best practices"
3. ChatGPT: "Какие лучшие практики для [должность]?"

**Что искать:**
- Методологии и фреймворки
- Примеры успешных кейсов
- Чеклисты качества
- Типичные ошибки

**Пример для копирайтера:**
- Формула AIDA (Attention, Interest, Desire, Action)
- Правило 5 секунд (захватить внимание)
- Чеклист: читаемость, CTA, эмоция

## ШАГ 6: СОЗДАНИЕ

**Цель:** Создать файлы агента.

### Что создаём:

**1. Файл правил** `.cursor/rules/[агент].mdc`

Это главный файл — "мозг" агента. Содержит:
- Роль (кто этот агент)
- Задачи (что делает)
- Инструкции (как делает)
- Правила (что можно/нельзя)

**2. Папка агента** `агенты/[АГЕНТ]/`

```
агенты/[АГЕНТ]/
├── _КОНТЕКСТ/
│   └── примеры/     ← Успешные кейсы для обучения
├── _ЛОГИ/           ← История работы
└── README.md        ← Описание агента
```

### Структура файла .mdc

```markdown
# [НАЗВАНИЕ АГЕНТА]

## РОЛЬ
Ты — [кто], [главная задача].
Команда для вызова: `@[имя]`

## КОНТЕКСТ
Папка агента: `агенты/[АГЕНТ]/`
Ключевые файлы: [список]

## ЗАДАЧИ
### Основные:
1. [Задача]
2. [Задача]

### НЕ входит:
- [Что НЕ делает]

## ИНСТРУКЦИИ
При получении задачи:
1. [Шаг]
2. [Шаг]
3. [Шаг]

## ПРАВИЛА РАБОТЫ
### Всегда:
- ✅ [Правило]

### Никогда:
- ❌ [Правило]

## ПРИМЕРЫ
### Пример 1: [Название]
**Запрос:** [Что попросили]
**Результат:** [Что сделал]
```

## ШАГ 7: ТЕСТИРОВАНИЕ

**Цель:** Проверить, что агент работает.

**3 типа тестовых задач:**

1. **Простая задача** — проверка базового функционала
   - Пример: "Напиши короткий пост о нашем продукте"

2. **Типичная задача** — обычная рабочая ситуация
   - Пример: "Создай серию из 3 постов для запуска курса"

3. **Сложная задача** — проверка границ
   - Пример: "Напиши пост для аудитории, которую не знаешь"

**После теста:**
- Собери обратную связь (что хорошо, что плохо)
- Улучши правила агента
- Повтори тест

---

## СИСТЕМА САМООБУЧЕНИЯ (КРИТИЧНО!)

Каждый агент должен **улучшаться** с каждым использованием.

### Как это работает:

**1. Запрос обратной связи**

После каждой задачи агент спрашивает:
> "Оцените результат:
> - ✅ Отлично
> - ⚠️ Нужны правки
> - ❌ Не то"

**2. Сохранение успешных кейсов**

При оценке "Отлично" — сохранить в `_КОНТЕКСТ/примеры/`:
- Исходный запрос
- Результат
- Почему сработало

**3. Логирование**

Каждое выполнение → запись в `_ЛОГИ/`:
```markdown
## [Дата] — [Тип задачи]
**Запрос:** [Что попросили]
**Результат:** [Что сделано]
**Оценка:** [✅ / ⚠️ / ❌]
**Уроки:** [Что учесть]
```

### Добавь в правила агента:

```markdown
## САМООБУЧЕНИЕ

### После каждой задачи:
1. Запросить оценку результата
2. При оценке "Отлично" → сохранить в `_КОНТЕКСТ/примеры/`
3. Логировать в `_ЛОГИ/[дата].md`

### Правило:
Если запрос имеет логику → сохранить в контекст.
```

## МЕХАНИКА ВОПРОСОВ (КРИТИЧНО!)

Агент должен **спрашивать**, когда непонятно.

### Когда спрашивать:
- Недостаточно контекста
- Неясная задача
- Отсутствуют данные
- Конфликт требований

### Как спрашивать:

**ПРАВИЛО:** 2-3 вопроса за раз, не больше!

**Плохо:**
```
Для выполнения задачи мне нужно знать:
1. Какой формат?
2. За какой период?
3. Какие метрики?
4. Кто аудитория?
5. Есть ли шаблон?
6. Какой дедлайн?
7. Нужны ли графики?
```

**Хорошо:**
```
Уточню:
1. Какой формат? (краткий / детальный)
2. За какой период? (неделя / месяц)

Остальное спрошу после ответа.
```

### Добавь в правила агента:

```markdown
## УТОЧНЯЮЩИЕ ВОПРОСЫ

При недостаточном контексте:
1. Определить, чего не хватает
2. Задать 2-3 вопроса (не больше!)
3. Дождаться ответа
4. При необходимости — уточнить ещё

**Важно:** Не перегружать вопросами!
```
