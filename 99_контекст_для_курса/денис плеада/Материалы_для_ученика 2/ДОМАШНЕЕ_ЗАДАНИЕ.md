## ЧТО ТЫ СОЗДАШЬ

**Telegram бот-помощник**, который:
- Принимает текстовые сообщения и сохраняет в папку Распределение
- Транскрибирует голосовые сообщения через AssemblyAI
- Обрабатывает файлы (документы, изображения)
- Сохраняет всё в структурированную папку для работы в Cursor

---

## ЗАЧЕМ ЭТО НУЖНО

**Проблема:** Нет ноутбука, а нужно сохранить важную мысль для Cursor.

**Решение:** Telegram бот, который принимает информацию 24/7 и складывает её в папку.

**Результат:** Все идеи, голосовые, файлы автоматически попадают в систему и не теряются.

## ЧЕК-ЛИСТ ВЫПОЛНЕНИЯ

### ☐ ШАГ 1: Создай Telegram бота (10 минут)

1. Открой Telegram
2. Найди **@BotFather**
3. Отправь команду `/newbot`
4. Придумай имя боту (например: "Мой Помощник Bot")
5. Придумай username (должен заканчиваться на `bot`, например: `my_helper_bot`)
6. **Скопируй API-токен** (это длинная строка вида `123456:ABC-DEF...`)
7. Сохрани токен в безопасном месте

**Проверка:** Бот создан, у тебя есть токен.

---

### ☐ ШАГ 2: Зарегистрируйся в AssemblyAI (5 минут)

1. Перейди на [assemblyai.com](https://www.assemblyai.com/)
2. Нажми **Sign Up**
3. Зарегистрируйся через почту
4. Подтверди email
5. Зайди в Dashboard → **API Keys**
6. **Скопируй API-ключ**
7. Сохрани ключ

**Бонус:** На счёт автоматически придёт **$50** (промокод от курса). Этого хватит на ~1000 транскрипций.

**Проверка:** У тебя есть API-ключ AssemblyAI.

---

### ☐ ШАГ 3: Узнай свой Telegram ID (2 минуты)

1. Открой Telegram
2. Найди **@userinfobot**
3. Нажми `/start`
4. Бот пришлёт твой **Telegram ID** (число вида `123456789`)
5. **Скопируй ID**

**Зачем:** Чтобы только ты мог использовать бота (защита от чужих).

**Проверка:** У тебя есть Telegram ID.

---

### ☐ ШАГ 4: Настрой бота в Cursor (20 минут)

1. Открой Cursor
2. Скачай **Материалы_для_ученика** из модуля 6
3. Распакуй папку в свой проект Cursor
4. Открой файл `config_example.py`
5. Заполни данные:

```python
# Telegram Bot
TELEGRAM_BOT_TOKEN = "ВАШ_ТОКЕН_ОТ_BOTFATHER"
ADMIN_TELEGRAM_ID = "ВАШ_TELEGRAM_ID"

# AssemblyAI
ASSEMBLYAI_API_KEY = "ВАШ_КЛЮЧ_ASSEMBLYAI"

# Папка для сохранения
SAVE_FOLDER = "_INBOX/_РАСПРЕДЕЛИ/"
```

6. Сохрани файл как `config.py` (без `_example`)
7. Открой терминал в Cursor
8. Установи зависимости:

```bash
pip install -r requirements.txt
```

**Проверка:** Файл `config.py` заполнен, зависимости установлены.

---

### ☐ ШАГ 5: Запусти бота (5 минут)

1. В терминале Cursor выполни:

```bash
python bot.py
```

2. Должна появиться надпись: **"Бот запущен. Ожидание сообщений..."**

**Проверка:** Бот работает, в терминале нет ошибок.

---

### ☐ ШАГ 6: Протестируй текстовые сообщения (3 минуты)

1. Открой Telegram
2. Найди своего бота (по username)
3. Напиши: **"Сохрани эту важную мысль для обучения"**
4. Бот ответит: **"✅ Сообщение сохранено"**
5. Проверь папку `_INBOX/_РАСПРЕДЕЛИ/` в Cursor
6. Там должен появиться файл `messages_YYYY-MM-DD.md` с текстом

**Проверка:** Текстовое сообщение сохранилось в файл.

---

### ☐ ШАГ 7: Протестируй голосовые сообщения (5 минут)

1. В Telegram запиши **голосовое сообщение** боту (10-30 секунд)
2. Скажи что-то вроде: *"Это тестовое голосовое сообщение для проверки транскрибации через AssemblyAI"*
3. Отправь голосовое боту
4. Подожди 10-20 секунд
5. Бот пришлёт **текстовую расшифровку**
6. Проверь папку `_INBOX/_РАСПРЕДЕЛИ/transcriptions/`
7. Там должен быть файл с расшифровкой

**Проверка:** Голосовое сообщение транскрибировалось в текст.

---

### ☐ ШАГ 8: Протестируй отправку файлов (3 минуты)

1. Отправь боту любой файл:
   - Документ (PDF, DOCX, TXT)
   - Или изображение (JPG, PNG)
2. Бот ответит: **"✅ Файл сохранен"**
3. Проверь папку `_INBOX/_РАСПРЕДЕЛИ/files/`
4. Файл должен быть там

**Проверка:** Файл сохранился.

## ЧАСТЫЕ ОШИБКИ И РЕШЕНИЯ

### ❌ Ошибка: "Module not found: telegram"

**Решение:**
```bash
pip install python-telegram-bot
```

---

### ❌ Ошибка: "Invalid token"

**Решение:**
- Проверь, что скопировал токен целиком
- Проверь, что нет лишних пробелов в `config.py`
- Создай нового бота через @BotFather

---

### ❌ Бот не отвечает на сообщения

**Решение:**
- Убедись, что `bot.py` запущен (терминал показывает "Бот запущен...")
- Проверь, что отправляешь сообщения СВОЕМУ боту
- Проверь интернет-соединение

---

### ❌ Голосовые не транскрибируются

**Решение:**
- Проверь API-ключ AssemblyAI
- Убедись, что на счёту есть баланс
- Проверь длину голосового (не больше 3 минут для теста)

---

### ❌ "Permission denied" при сохранении файлов

**Решение:**
- Проверь, что папка `_INBOX/_РАСПРЕДЕЛИ/` существует
- Создай папку вручную через Cursor
- Проверь права доступа к папке

## ВАЖНО: БЕЗОПАСНОСТЬ API-КЛЮЧЕЙ

**⚠️ КРИТИЧНО:**

1. **НИКОМУ** не передавай API-ключи
2. **НЕ публикуй** ключи в открытых репозиториях
3. **НЕ делись** скриншотами с открытыми ключами
4. **Храни** ключи в `config.py` (файл должен быть в `.gitignore`)

**Почему:**
Если кто-то получит твой API-ключ:
- Сможет управлять твоим ботом
- Видеть все сообщения
- Тратить твой баланс AssemblyAI
- Использовать бота в своих целях

**Правило:** API-ключ = пароль от банковской карты.

## ЛОКАЛЬНАЯ РАБОТА VS СЕРВЕР

### Что важно понять:

**Локальная работа:**
- ✅ Бесплатно
- ✅ Просто настроить
- ❌ Работает только когда компьютер включён
- ❌ Telegram хранит сообщения 24 часа (если бот выключен дольше — теряются)

**Сервер (следующий уровень):**
- ✅ Работает 24/7
- ✅ Не теряет сообщения
- ❌ Стоит ~600₽/месяц
- ❌ Нужна настройка

**Рекомендация:** Начни с локальной работы. Если бот станет незаменимым — переноси на сервер.

**Лайфхак:** Если не успел включить бота за 24 часа — переслать сообщение самому себе в Telegram (там оно хранится вечно).
