# ТЗ: Telegram бот-помощник для обучения

> **Модуль 6: API Интеграции**
> **Курс:** CURSOR — Сотрудники под твой бизнес за вечер

---

## Цель

Создать Telegram бота-помощника, который:
- Принимает сообщения, файлы, голосовые из Telegram
- Сохраняет всё в папку "Распределение" для работы в Cursor
- Транскрибирует голосовые и видео через AssemblyAI
- Работает локально (только когда ноутбук включен)

**Результат:** Любая информация из Telegram автоматически попадает в твой проект Cursor.

---

## Функции бота (8 пунктов)

### 1. Сохранение текстовых сообщений
- Принимает текст → сохраняет в `Распределение/сообщения/YYYY-MM-DD_HH-MM-SS.md`
- Формат файла: дата, время, текст, метаданные пользователя

### 2. Транскрибация голосовых сообщений
- Принимает голосовое → отправляет в AssemblyAI → получает транскрипт
- Сохраняет в `Распределение/транскрипты/YYYY-MM-DD_HH-MM-SS_voice.md`

### 3. Транскрибация видео файлов
- Принимает видео (MP4, MOV, AVI) → извлекает аудио → транскрибирует
- Сохраняет в `Распределение/транскрипты/YYYY-MM-DD_HH-MM-SS_video.md`

### 4. Сохранение документов
- Принимает документы (PDF, DOCX, TXT)
- Сохраняет в `Распределение/документы/` с метаданными (название, размер, тип)

### 5. Сохранение изображений
- Принимает изображения (JPG, PNG, WEBP)
- Сохраняет в `Распределение/изображения/` с timestamp в названии

### 6. Парсинг ссылок (опционально)
- Определяет ссылки в тексте → парсит заголовок страницы
- Сохраняет в `Распределение/ссылки/` с URL и заголовком

### 7. Команды бота
| Команда | Что делает |
|---------|-----------|
| `/start` | Приветствие и инструкция |
| `/help` | Справка по командам |
| `/status` | Показывает путь к папке Распределение |
| `/sync` | Статистика: сколько сообщений обработано |
| `/log` | Последние 10 записей из лога бота |
| `/list` | Список последних сохраненных файлов |

### 8. Синхронизация при запуске
- При запуске бота — получает все накопившиеся сообщения из Telegram
- Обрабатывает их последовательно
- Отправляет отчёт: "Обработано X сообщений за время простоя"

## Механика синхронизации

### Как работает Telegram
- Telegram хранит непрочитанные сообщения на сервере **24 часа**
- Когда бот запускается — он получает ВСЕ накопившиеся сообщения
- Если бот был выключен >24 часов — старые сообщения теряются

### Механика бота при запуске
1. Бот запускается (`python bot.py`)
2. Telegram отдаёт все непрочитанные сообщения (очередь)
3. Бот обрабатывает их последовательно: сохраняет файлы, транскрибирует голосовые
4. Отправляет отчёт в чат: "Обработано X сообщений за время простоя"

### Отметка обработанных сообщений
- Файл `processed.json` хранит ID обработанных сообщений
- При обработке сообщения — ID добавляется в файл
- При повторном получении того же сообщения — пропускается

### Логирование
- Файл `bot.log` записывает все действия бота
- Формат: `[ДАТА ВРЕМЯ] [ТИП] Сообщение`
- Команда `/log` показывает последние 10 записей

### Пример вывода команды `/sync`

```

Последний запуск: 2026-01-11 14:30:15
Обработано сообщений: 47
Сохранено файлов: 23
Транскрибировано: 5

Очередь: 0 сообщений ожидают обработки
```

1. **Бот работает локально** — только когда ноутбук включен
