# 📟 WORKBOOK: МОДУЛЬ 3 — СИСТЕМА ПОРЯДКА (Context-First)

> **CURSOR: Экзоскелет для ума**
> Этот файл — пульт управления для построения архитектуры твоего бизнеса.
> Мы не будем натягивать твой бизнес на шаблон. Мы научим AI понимать твой бизнес, а потом построим под него идеальный дом.

---

## 💻 ВАЖНО: ГОРЯЧИЕ КЛАВИШИ (MAC)
- Везде используем **Cmd** (⌘).
- Открыть чат Composer: `Cmd+I`.
- Добавить файл в контекст: `@файл`.

---

## 🟢 ШАГ 1: СБОР ДАННЫХ (FEEDING)

Прежде чем строить, нужно дать AI "попробовать" твой бизнес на вкус.

**1. Создай папку-приемник:**
Скопируй промпт ниже и нажми `Cmd+I`:

```text
Создай в корне проекта папку "00_Inbox".
Внутри создай README.md с текстом: "Inbox для хаоса. Сюда попадают файлы перед сортировкой."
```

**2. Наполни папку (Сделай это руками сейчас):**
Перетащи в папку `00_Inbox` ключевые файлы твоего проекта:
*   [ ] Оргструктура (если есть)
*   [ ] Прайс-лист / Описание продукта
*   [ ] Любые презентации о компании
*   [ ] Брифы / Стратегии

*(Если файлов нет — ничего страшного, пойдем через сайт).*

---

## 🟢 ШАГ 2: ЦИФРОВОЙ СЛЕПОК (PROFILING)

Теперь AI изучит тебя через файлы и интернет.

**Открой Composer (`Cmd+I`) и отправь этот промпт:**
*(Замени ссылки на свои)*

```text
@00_Inbox (папка)

Я хочу, чтобы ты глубоко понял мой контекст.
1. Прочитай все файлы в папке "00_Inbox".
2. Изучи мой сайт/соцсети (используй поиск):
   - Сайт: [ВСТАВЬ ССЫЛКУ]
   - Соцсети/Канал: [ВСТАВЬ ССЫЛКУ]
   - Обо мне: [Кратко кто ты, если нет ссылок]

Сформируй "Профиль Проекта":
- Суть бизнеса (в одном предложении).
- Ключевые продукты/услуги.
- Аудитория.
- Какие отделы/зоны ответственности должны быть.

Не предлагай структуру папок пока. Просто напиши "Профиль Проекта", чтобы я подтвердил, что ты правильно всё понял.
```

---

## 🟢 ШАГ 3: ПРОЕКТИРОВАНИЕ (ARCHITECTING)

Когда AI подтвердил, что понял бизнес, пусть предложит структуру.

**Отправь этот промпт:**

```text
На основе "Профиля Проекта", который мы утвердили, предложи структуру папок для моего проекта.

Используй логику "Системы Папок" (как в файле @ШАБЛОН_БИЗНЕС_СТРУКТУРЫ.md), но адаптируй названия под МЕНЯ.

Правила:
1. Корневая папка всегда одна.
2. Внутри 5-7 основных папок (нумерация 01_, 02_...).
3. Обязательно должна быть папка "99_Системное" (для промптов и настроек).
4. Обязательно "00_Inbox".

Выведи предложенную структуру списком. Я хочу утвердить её перед созданием.
```

---

## 🟢 ШАГ 4: СТРОИТЕЛЬСТВО (BUILDING)

Если структура тебе нравится — даем команду на стройку.

**Отправь промпт:**

```text
Структура утверждена. Действуй:

1. Создай эту структуру папок.
2. ВНУТРИ КАЖДОЙ папки (и подпапок) сразу создай файл README.md.
3. В README.md используй профессиональный шаблон:
   - 🎯 О папке (зачем нужна в моем бизнесе)
   - 📂 Содержимое (что тут хранить)
   - ⚙️ Как использовать

Сделай это сейчас.
```

---

## 🟢 ШАГ 5: ЗАСЕЛЕНИЕ (DISTRIBUTION)

Теперь перенесем файлы из хаоса в порядок.

**Отправь промпт:**

```text
Теперь посмотри на файлы, которые лежат в @00_Inbox.
Учитывая новую структуру и контекст бизнеса:

1. Предложи, куда переместить каждый файл.
2. Если файл никуда не подходит — скажи об этом.
3. После моего подтверждения — перемести файлы.
```
