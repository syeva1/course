# Структура проекта для AI-агентов (Claude Code)

Источник: @misha_davai_po_novoi, 10 марта 2026

## Структура

```
project/
├── CLAUDE.md              # Главный контекст (= .cursorrules)
├── README.md
├── docs/
│   ├── architecture.md    # Архитектура проекта
│   ├── decisions/         # Записи о принятых решениях
│   └── runbooks/          # Пошаговые инструкции
├── .claude/
│   ├── settings.json
│   ├── hooks/             # Автоматические проверки
│   └── skills/            # Скиллы для повторяющихся задач
│       ├── code-review/SKILL.md
│       ├── refactor/SKILL.md
│       └── release/SKILL.md
├── tools/
│   ├── scripts/
│   └── prompts/
└── src/
    └── api/
        └── CLAUDE.md      # Вложенный контекст для подпапки
```

## Ключевые принципы

1. **CLAUDE.md** — главный файл, агент читает при каждом запуске. Только самое важное, коротко и по делу. Не свалка.

2. **Skills** — готовые инструкции для повторяющихся задач. Пишется один раз, агент сам подтягивает когда нужно.

3. **Hooks** — скрипты, которые запускаются автоматически (до/после команд). В отличие от инструкций, хуки нельзя проигнорировать. Пример: проверка текста на нейросетевые маркеры.

4. **Вложенные контексты** — в каждой папке свой CLAUDE.md, агент подтягивает ближайший к текущей задаче.

5. **docs/** — архитектура, решения, инструкции. Агент может читать для контекста.

6. **tools/** — промпты и скрипты, которые не привязаны к конкретному скиллу.

## Аналогии с Cursor

| Claude Code | Cursor |
|-------------|--------|
| CLAUDE.md | .cursorrules / .mdc правила |
| .claude/skills/ | Agents в Cursor |
| .claude/hooks/ | (нет прямого аналога — ручные проверки) |
| Вложенные CLAUDE.md | .mdc с globs по папкам |

## Применение для курса АССИСТ+

- Модуль 02 (Контекст): структура проекта
- Модуль 05 (Автоматизация): hooks как автопроверки
- Модуль 07 (Агенты): skills как переиспользуемые агенты
