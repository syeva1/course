## Система мульти-агентов

### Что вы узнаете

1. Зачем нужна команда агентов вместо одного
2. Как устроен CEO-агент (оркестратор)
3. Как работает цепочка: planner → checker → writer → checker → reviewer
4. Как организовать параллельную работу
5. Инструменты: ccpm, agtx
6. Как логировать и дебажить мульти-агентные системы

### Порядок прохождения

1. **Откройте презентацию** — `презентация.html`
2. **Пролистайте все 17 слайдов** — каждый раскрывает отдельную тему
3. **Изучите шпаргалку** — `шпаргалка.md` для быстрого повторения
4. **Посмотрите материалы** — схема и шаблон цепочки
5. **Выполните практику** — `практика.md`

### CEO-агент: как это работает

CEO-агент — это главный агент, который:
- Получает задачу от человека
- Разбивает её на подзадачи
- Раздаёт подзадачи специализированным агентам
- Контролирует качество результатов
- Собирает финальный результат

### Content Factory: пример цепочки

```
Planner → Plan-Checker → Writer → Text-Checker → Final-Reviewer
```

Каждый агент:
- Читает файл от предыдущего (вход)
- Делает свою работу
- Пишет результат в файл (выход)

### Промежуточные файлы

```
project/
├── plan.md          ← что писать
├── plan-review.md   ← замечания к плану
├── draft.md         ← черновик
├── text-review.md   ← замечания к тексту
├── final.md         ← финальная версия
└── log.json         ← логи всех агентов
```

### Инструменты

- **ccpm** — Claude Code Project Manager: PRD → эпики → Issues → агенты
- **agtx** — канбан-доска в терминале для отслеживания задач

### Советы

- Начинайте с 2–3 агентов, не больше
- Всегда добавляйте проверщика
- Логируйте всё: вход, выход, время, токены
- Тестируйте каждого агента отдельно
