# Модуль 09 — Шпаргалка

## Мульти-агентные системы

### Ключевые концепции

| Термин | Значение |
|--------|----------|
| CEO-агент | Главный агент-оркестратор, раздаёт задачи |
| Цепочка | Последовательность агентов: A → B → C |
| Параллелизм | Несколько цепочек работают одновременно |
| Промежуточные файлы | plan.md, draft.md, final.md — передача данных |
| Логирование | Запись действий каждого агента |

### Типичная цепочка (Content Factory)

```
Planner → Plan-Checker → Writer → Text-Checker → Final-Reviewer
```

### Инструменты

| Инструмент | Назначение |
|------------|------------|
| ccpm | PRD → эпики → Issues → агенты |
| agtx | Канбан-доска в терминале |
| Agent Lightning | Автоулучшение промптов (Microsoft) |

### Правила мульти-агентных систем

1. **Один агент = одна роль** — не мешайте обязанности
2. **Всегда проверщик** — после каждого этапа
3. **Файлы, не контекст** — данные через файлы, не через общий промпт
4. **Логируйте всё** — вход, выход, время, токены
5. **Начинайте с малого** — 2–3 агента, потом масштабируйте
6. **Тестируйте отдельно** — каждый агент должен работать сам по себе

### Структура проекта

```
project/
├── prompts/         ← промпты агентов
│   ├── planner.md
│   ├── checker.md
│   └── writer.md
├── output/          ← результаты
│   ├── plan.md
│   ├── draft.md
│   └── final.md
└── logs/            ← логи
    └── log.json
```

### Быстрый старт

```bash
# 1. Создайте папку проекта
mkdir my-agents && cd my-agents

# 2. Создайте промпты
mkdir prompts output logs

# 3. Напишите промпт для первого агента
cat > prompts/planner.md << 'EOF'
Ты — планировщик. Создай план статьи на основе темы.
Вход: тема.txt
Выход: output/plan.md
EOF

# 4. Запустите
claude -p "$(cat prompts/planner.md)" < тема.txt > output/plan.md
```
