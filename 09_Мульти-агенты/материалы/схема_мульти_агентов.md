# Схема мульти-агентной системы

## Общая архитектура

```
                    ┌─────────────────┐
                    │   👤 Человек     │
                    │  (ставит задачу) │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │  🧠 CEO-агент    │
                    │  (оркестратор)   │
                    └────────┬────────┘
                             │
            ┌────────────────┼────────────────┐
            │                │                │
   ┌────────▼──────┐ ┌──────▼────────┐ ┌─────▼───────┐
   │ Воркфлоу #1   │ │ Воркфлоу #2   │ │ Воркфлоу #3 │
   │ (контент)     │ │ (email)       │ │ (аналитика) │
   └────────┬──────┘ └──────┬────────┘ └─────┬───────┘
            │                │                │
        Цепочка          Цепочка          Цепочка
        агентов          агентов          агентов
```

## Цепочка агентов (детально)

```
┌──────────┐    plan.md    ┌──────────────┐   ok/fix   ┌──────────┐
│ Planner  │──────────────▶│ Plan-Checker │──────────▶│  Writer  │
└──────────┘               └──────────────┘            └─────┬────┘
                                                             │
                                                          draft.md
                                                             │
                                                     ┌───────▼──────┐
                                                     │ Text-Checker  │
                                                     └───────┬──────┘
                                                             │
                                                        review.md
                                                             │
                                                    ┌────────▼───────┐
                                                    │ Final-Reviewer │
                                                    └────────┬───────┘
                                                             │
                                                          final.md
```

## Способы коммуникации

| Способ | Описание | Когда использовать |
|--------|----------|--------------------|
| Файлы | Агент пишет результат в файл, следующий читает | Основной способ |
| Промпт | CEO передаёт инструкции через промпт | Задание задачи |
| Статус | Агент сообщает «готово» или «ошибка» | Оркестрация |
| Git | Результаты коммитятся в репозиторий | Версионирование |

## Паттерны

### 1. Последовательная цепочка
```
A → B → C → D
```
Простейший вариант. Каждый агент ждёт предыдущего.

### 2. Параллельные цепочки
```
A → B → C
A → D → E
A → F → G
```
CEO запускает несколько цепочек одновременно.

### 3. Fan-out / Fan-in
```
     ┌→ B ─┐
A ───┼→ C ─┼──→ F (собирает результаты)
     └→ D ─┘
```
Одна задача разбивается, результаты собираются.
