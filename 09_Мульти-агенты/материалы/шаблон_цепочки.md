# Шаблон мульти-агентной цепочки

## Заполните для своего проекта

### Название проекта
_Например: «Генерация контента для блога»_

### Цель
_Что должна делать цепочка от начала до конца?_

### Агенты

#### Агент 1: ___________
- **Роль:** _что делает_
- **Вход:** _какой файл читает_
- **Выход:** _какой файл создаёт_
- **Промпт:**
```
Ты — [роль]. Твоя задача — [что делать].
Прочитай файл [вход].
Создай файл [выход] с результатом.
Формат: [описание формата]
```

#### Агент 2: ___________
- **Роль:** _что делает_
- **Вход:** _какой файл читает_
- **Выход:** _какой файл создаёт_
- **Промпт:**
```
Ты — [роль]. Твоя задача — [что делать].
Прочитай файл [вход].
Создай файл [выход] с результатом.
```

#### Агент 3 (проверщик): ___________
- **Роль:** _проверка качества_
- **Вход:** _файл от предыдущего агента_
- **Выход:** _review-файл или исправленная версия_
- **Промпт:**
```
Ты — проверщик. Оцени качество файла [вход].
Если всё хорошо — напиши "OK" в [выход].
Если есть замечания — перечисли их и предложи исправления.
```

_(добавьте больше агентов по необходимости)_

### Схема цепочки

```
Агент 1 → Агент 2 → Агент 3 (проверщик) → ...
  ↓           ↓           ↓
[файл1]    [файл2]    [review]
```

### Чеклист перед запуском

- [ ] Все промпты написаны и протестированы отдельно
- [ ] Входы и выходы не пересекаются
- [ ] Есть хотя бы один проверщик в цепочке
- [ ] Логирование настроено (время, токены, статус)
- [ ] Цепочка протестирована на одном примере

### Пример запуска (bash)

```bash
# Последовательный запуск
claude -p "$(cat prompts/agent1.md)" && \
claude -p "$(cat prompts/agent2.md)" && \
claude -p "$(cat prompts/checker.md)"

# Параллельный запуск (несколько задач)
for task in tasks/*.txt; do
  (
    claude -p "$(cat prompts/agent1.md)" < "$task" > "output/$(basename $task .txt)-plan.md"
    claude -p "$(cat prompts/agent2.md)" < "output/$(basename $task .txt)-plan.md" > "output/$(basename $task .txt)-draft.md"
  ) &
done
wait
echo "Все задачи завершены"
```
