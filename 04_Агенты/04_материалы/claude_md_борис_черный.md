## Шаблон (адаптирован для Cursor)

```
## Оркестрация рабочего процесса

1. **Режим планирования по умолчанию** — для ЛЮБОЙ нетривиальной задачи (3+ шагов):
   - Сначала составь план
   - Если что-то идёт не так — ОСТАНОВИСЬ и перепланируй

2. **Стратегия маленьких шагов** — разбивай задачу на атомарные шаги.
   Каждый шаг — проверяемый результат.

3. **Цикл самосовершенствования** — после ЛЮБОГО замечания обновляй lessons.md.
   Пиши правила, предотвращающие повторение ошибки.

4. **Проверка перед завершением** — никогда не отмечай задачу выполненной, не доказав, что она работает.
   Спроси себя: «Одобрил бы это ведущий инженер?»

5. **Требуй элегантности** — для нетривиальных изменений спроси: «есть ли более элегантный способ?»

6. **Автономное исправление багов** — получил баг → просто исправь. Не проси вести за ручку.

## Управление задачами

- Сначала план в tasks/todo.md
- Согласуй с пользователем перед реализацией
- Отслеживай прогресс
- Фиксируй уроки в tasks/lessons.md

## Ключевые принципы

- Простота прежде всего
- Никакой лени — ищи корневые причины
- Минимальное воздействие на существующий код
- Один коммит = одна логическая единица
```

## Как применить в Cursor

1. Скопируй текст выше в `.cursor/rules/workflow.mdc`
2. Создай файл `tasks/todo.md` в корне проекта
3. Создай файл `tasks/lessons.md`
4. При каждой ошибке — дописывай урок в lessons.md
5. Cursor будет автоматически следовать этим правилам
