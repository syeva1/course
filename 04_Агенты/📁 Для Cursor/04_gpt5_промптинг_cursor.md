## Ключевые инсайты для пользователей Cursor

### 1. Не надо просить «будь тщательным»

Старые модели нуждались в подталкивании:
```
❌ Будь ТЩАТЕЛЬНЫМ. Убедись, что у тебя ПОЛНАЯ картина перед ответом.
```

GPT-5 и так тщательный — такая инструкция заставляет его **перестараться**: бесконечно искать контекст вместо работы.

```
✅ Если выполнил часть задачи, но не уверен — собери больше информации
   перед завершением. Старайся найти ответ сам, а не спрашивать.
```

### 2. Пусть AI действует, а не спрашивает

Cursor обнаружил, что GPT-5 слишком часто спрашивает «Продолжить?». Решение:

```
Ты предлагаешь изменения в коде, которые пользователь может принять
или отклонить. Это значит:
(a) можешь действовать проактивно — пользователь всегда может отклонить
(b) код должен быть читаемым для быстрого ревью
Не спрашивай, делать ли дальше — делай и предложи на одобрение.
```

### 3. Управляй «жадностью» AI

**Хочешь быстрый ответ** (простая задача):
```xml
<context_gathering>
- Глубина поиска: минимальная
- Отвечай максимально быстро, даже если не на 100% точно
- Максимум 2 вызова инструментов
- Если нужно больше — скажи, что нашёл, и спроси
</context_gathering>
```

**Хочешь полный результат** (сложная задача):
```xml
<persistence>
- Продолжай, пока задача полностью не решена
- Не останавливайся при неопределённости — выбери разумный подход
- Не спрашивай подтверждения — просто делай и документируй решения
</persistence>
```

### 4. Self-reflection для крутых результатов

Лайфхак от OpenAI для создания приложений с нуля:

```xml
<self_reflection>
- Сначала придумай критерии оценки (5-7 категорий)
- Не показывай критерии пользователю — они для тебя
- Используй критерии для внутренней итерации
- Если результат не проходит по всем категориям — начни заново
</self_reflection>
```

### 5. Противоречия в промптах — убийца

GPT-5 **буквально** следует инструкциям. Если в промпте есть противоречия — он потратит всю «мощность» на попытки их разрешить.

**Проверяй промпты:** нет ли в них взаимоисключающих правил. GPT-5 можно использовать как мета-промптер:

```
Вот мой промпт: [ПРОМПТ]
Я хочу, чтобы AI [ЖЕЛАЕМОЕ ПОВЕДЕНИЕ], но он [ДЕЛАЕТ ДРУГОЕ].
Какие минимальные правки исправят это?
```

## Структура промптов для Cursor (XML-теги)

Команда Cursor использует XML-теги для структурирования:

```xml
<code_editing_rules>
  <guiding_principles>
    - Каждый компонент модульный и переиспользуемый
    - Единая дизайн-система (цвета, типографика, отступы)
    - Простота: маленькие компоненты, минимум сложности
  </guiding_principles>

  <ui_ux_best_practices>
    - 4-5 размеров шрифта максимум
    - 1 нейтральный цвет + 2 акцентных
    - Отступы кратны 4 (padding: 4, 8, 12, 16...)
    - Скелетоны при загрузке данных
    - Hover-эффекты для кликабельных элементов
  </ui_ux_best_practices>
</code_editing_rules>
```
