## Разница

| Промпт | Скилл |
|--------|-------|
| Одноразовый текст | Переиспользуемый компонент |
| Пишешь каждый раз заново | Написал один раз — работает всегда |
| Нет структуры | Инструкции + примеры + правила |
| Для одной задачи | Для категории задач |
| Живёт в чате | Живёт в файле (.mdc) |

## Что такое скилл

Скилл = набор инструкций + примеров + правил для конкретной задачи.

**Промпт:** «Напиши пост для Telegram, 500 символов, дружелюбный тон»

**Скилл:**
```
---
description: "Написание постов для Telegram-канала"
globs: ["content/**"]
---

## Роль
Ты — SMM-копирайтер компании АССИСТ+.

## Правила
- Длина: 500-800 символов
- Тон: дружелюбный, экспертный
- Формат: эмодзи-заголовок → польза → CTA → хештеги
- Прочитай config/style-guide.md перед написанием
- Посмотри примеры в config/examples/posts/

## Структура поста
1. Эмодзи + жирный заголовок (вопрос или утверждение)
2. Основной текст (3-5 абзацев, каждый 1-2 предложения)
3. Призыв к действию
4. 2-3 хештега

## Запрещено
- Кликбейт
- «Уникальный», «инновационный»
- Более 5 эмодзи

## Примеры хорошего результата
[см. config/examples/posts/]
```

## Как создать скилл для Cursor

### Шаг 1: Определи задачу
Что ты делаешь регулярно? Пишешь посты? Ревьюишь код? Отвечаешь клиентам?

### Шаг 2: Создай .mdc файл
`.cursor/rules/название_скилла.mdc`

### Шаг 3: Структура скилла
```
---
description: "Когда активировать"
globs: ["путь/к/файлам/**"]
---

## Роль (кто ты)
## Задача (что делать)
## Правила (как делать)
## Формат (что на выходе)
## Примеры (как должно выглядеть)
## Запрещено (чего не делать)
```

### Шаг 4: Добавь примеры
Примеры — самая важная часть. Один пример > 100 слов инструкций.

## 10 идей для скиллов

1. **Постинг** — пишет посты в стиле бренда
2. **Ревью кода** — проверяет и комментирует
3. **Email** — пишет письма клиентам
4. **Документация** — документирует код/процессы
5. **Тестирование** — пишет тесты
6. **Рефакторинг** — улучшает существующий код
7. **Перевод** — переводит с сохранением стиля
8. **Аналитика** — анализирует данные, делает выводы
9. **FAQ** — генерирует ответы на вопросы клиентов
10. **Отчёты** — создаёт отчёты из данных
