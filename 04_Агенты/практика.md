# Практика — Модуль 04. Агенты и скиллы

## Задание 1. Первый .mdc файл (повтори по инструкции)

1. Создай папку `.cursor/rules/` в проекте
2. Создай файл `копирайтер.mdc` с ролью, тоном, аудиторией, правилами
3. Создай текстовый файл и попроси AI написать текст для сайта
4. Проверь — AI пишет в заданном тоне? Соблюдает правила?

Результат: .mdc файл работает — AI пишет тексты в нужном стиле без дополнительных инструкций в промпте.

---

## Задание 2. Аналитик + тестовые данные (повтори по инструкции)

1. Создай файл `аналитик.mdc` с форматом ответа (вывод → цифры → рекомендации)
2. Создай CSV-файл с любыми бизнес-данными (продажи, расходы, клиенты)
3. Попроси AI проанализировать: `@File данные.csv Проанализируй`
4. Проверь формат ответа — соответствует правилам из .mdc?

Результат: Аналитик работает по шаблону и даёт структурированный ответ.

---

## Задание 3. Тройной прогон (сделай сам)

1. Выбери задачу: написать текст для email-рассылки (или любую другую важную задачу)
2. Выполни её в Claude Sonnet (переключи модель внизу Composer)
3. Переключи на GPT-4o → выполни ту же задачу
4. Сравни оба результата
5. Запиши в файл `сравнение_моделей.md`:
   - Какая модель что сделала лучше
   - Какую бы выбрал для этой задачи
   - Почему

Результат: Ты видишь разницу между моделями на реальной задаче и умеешь осознанно выбирать.

---

## Задание 4. lessons.md — обучение на ошибках (сделай сам)

1. Создай файл `lessons.md` в корне проекта
2. Дай своему копирайтеру 5 разных задач:
   - Текст для главной страницы
   - Пост для соцсетей
   - Описание услуги
   - Email клиенту
   - Текст для кнопки CTA
3. После каждой задачи оцени: что не так? Что бы исправил?
4. Запиши каждую ошибку в lessons.md (проблема → решение)
5. Обнови `копирайтер.mdc` с учётом ошибок
6. Повтори одну из задач — стало лучше?

Результат: Ты видишь как агент улучшается через lessons.md.

---

## Задание 5. Библиотека агентов (творческое)

Создай минимум 3 новых .mdc файла для своего бизнеса. Возьми за основу готовые из `материалы/готовые_mdc/` или придумай свои.

Идеи:
- Менеджер по продажам — составляет КП, отвечает на возражения
- Планировщик — разбивает задачи на шаги, расставляет приоритеты
- Ревьюер — проверяет тексты/код на ошибки и несоответствия
- Исследователь — ищет информацию, составляет обзоры
- Переводчик — переводит с учётом контекста бизнеса

Для каждого агента:
1. Создай .mdc файл
2. Протестируй на 2-3 задачах
3. Запиши ошибки в lessons.md
4. Обнови .mdc

Результат: Библиотека из 5+ агентов, протестированных на реальных задачах. Commit и push на GitHub.
