# 65 скиллов для продакт-менеджера в Claude Code / Cursor

> Источник: [github.com/phuryn/pm-skills](https://github.com/phuryn/pm-skills) | Канал: @misha_davai_po_novoi

## Что это

Набор из **65 скиллов** и **36 цепочек команд** для продакт-менеджерской работы. Работает с Claude Code, Cowork, Cursor и другими AI-инструментами.

Каждый скилл — это готовый фреймворк из книг Терезы Торрес, Марти Кагана, Альберто Савойи. Не просто текст, а **пошаговый процесс**.

## 8 плагинов (категорий)

### pm-product-discovery — Исследование продукта
- `/discover` — генерация идей → допущения → приоритизация → эксперименты
- Opportunity Solution Tree (дерево возможностей)
- Дизайн экспериментов
- Customer interview guide

### pm-product-strategy — Стратегия продукта
- `/strategy` — видение + бизнес-модель + позиционирование
- Lean Canvas, SWOT, PESTLE, Porter's Five Forces
- Стратегия монетизации и ценообразования
- Матрица Ансоффа

### pm-market-research — Исследование рынка
- Анализ конкурентов
- Размер рынка (TAM/SAM/SOM)
- Анализ трендов

### pm-data-analytics — Аналитика
- North Star Metric
- A/B тестирование
- Когортный анализ
- Воронки конверсии

### pm-execution — Исполнение
- `/write-prd` — написание PRD (Product Requirements Document)
- OKR, роадмапы
- Приоритизация (RICE, MoSCoW, Kano)
- User stories

### pm-go-to-market — Запуск
- `/plan-launch` — план запуска продукта
- Стратегия выхода на рынок
- Beta-программа

### pm-marketing-growth — Маркетинг и рост
- Позиционирование
- Naming (название продукта)
- Growth-стратегии

### pm-toolkit — Инструменты
- NDA, Privacy Policy
- Ревью резюме
- Шаблоны документов

## Как установить

### В Cursor (через скиллы)
Скопируй нужные `SKILL.md` файлы в `.cursor/skills/` своего проекта.

### В Claude Code
```bash
claude plugin marketplace add phuryn/pm-skills
claude plugin install pm-product-strategy@pm-skills
# ... и остальные плагины
```

## Почему это круто

Это пример того, как **скиллы** работают на практике. Не один промпт на все случаи, а 65 специализированных инструментов, каждый под свою задачу.

Команды связаны между собой — после одной агент предложит следующую подходящую. Это и есть **workflow из скиллов**.

## Применимость

Даже если ты не продакт-менеджер, этот набор показывает **паттерн**: как разбить сложную работу на скиллы и связать их в цепочки. Тот же подход работает для маркетолога, рекрутера, аналитика — кого угодно.
