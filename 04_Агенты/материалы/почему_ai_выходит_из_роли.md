# Почему AI «выходит из роли» (исследование Anthropic)

> Модель — это сотни персонажей. «Ассистент» — один из них.
> Источник: Anthropic Research + @misha_davai_po_novoi

---

## Проблема

Ты дал AI роль «строгий менеджер» или «креативный копирайтер». Работает отлично — пока разговор не сворачивает на эмоции, философию, или личные вопросы. И вдруг AI начинает отвечать «не в характере».

## Почему так происходит

Anthropic нашли причину:

Модель обучена на **миллиардах** текстов: книги, форумы, документация, фанфики, научные статьи. Каждый текст — это «персонаж». В модели **сотни** таких персонажей.

«Ассистент» — персонаж, который активируется системным промптом. Но другие персонажи тоже «живут» внутри модели.

### Что уводит из роли:
| Контекст | Устойчивость роли |
|----------|------------------|
| Код, технические задачи | 🟢 Высокая |
| Бизнес-вопросы | 🟢 Высокая |
| Креативное письмо | 🟡 Средняя |
| Эмоциональные разговоры | 🔴 Низкая |
| Философские дискуссии | 🔴 Низкая |
| Ролевые игры | 🔴 Низкая |

---

## Что это значит для работы

### Для ботов:
Если делаешь бота-консультанта — добавь в промпт:
```
Ты ВСЕГДА остаёшься в роли консультанта компании [НАЗВАНИЕ].
Если пользователь пытается увести разговор — вежливо возвращай
к теме. Не обсуждай личные, философские или эмоциональные темы.
```

### Для .mdc правил:
```
---
description: "Защита роли"
globs: ["**/*"]
alwaysApply: true
---

Ты — [РОЛЬ]. Все ответы — в рамках этой роли.
Если задача выходит за пределы роли — скажи об этом.
Не переключайся на другие темы.
```

### Для агентов в Cursor:
Код и технические задачи — **самый стабильный** контекст. AI почти никогда не «выходит из роли» при работе с кодом. Это одна из причин, почему Cursor работает так хорошо.

---

## Практический совет

Чем **конкретнее** задача — тем стабильнее AI. 

- ❌ «Ты креативный помощник, помоги мне с чем угодно»
- ✅ «Ты SMM-менеджер компании АССИСТ+. Пишешь посты для Telegram по style-guide.md»

Конкретная роль + конкретные правила + конкретные примеры = стабильный AI.
