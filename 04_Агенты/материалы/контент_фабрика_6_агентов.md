# Контент-фабрика: 6 агентов в цепочке

> Реальная система Миши Попова для создания контента.
> 5 статей параллельно, каждая — за 1-2 часа.
> Источник: @misha_davai_po_novoi

---

## 6 агентов

| # | Агент | Что делает | Вход → Выход |
|---|-------|-----------|--------------|
| 1 | **Скаут** | Ищет тренды и темы | Задание → список тем |
| 2 | **Аналитик** | Исследует тему, собирает факты | Тема → research.md |
| 3 | **Писатель** | Создаёт черновик | research.md → draft.md |
| 4 | **Редактор** | Правит текст, улучшает подачу | draft.md → edited.md |
| 5 | **Дизайнер** | Создаёт промпты для картинок | edited.md → visuals.md |
| 6 | **Публишер** | Форматирует для платформы | всё → final.md |

## Как реализовать в Cursor

### Шаг 1: Структура проекта

```
content-factory/
├── README.md
├── .cursor/rules/
│   ├── scout.mdc
│   ├── analyst.mdc
│   ├── writer.mdc
│   ├── editor.mdc
│   ├── designer.mdc
│   └── publisher.mdc
├── input/
│   └── brief.md         # Задание
├── output/
│   ├── research.md
│   ├── plan.md
│   ├── draft.md
│   ├── edited.md
│   ├── visuals.md
│   └── final.md
└── config/
    ├── style-guide.md    # Редполитика
    └── examples/         # Примеры хороших текстов
```

### Шаг 2: Пример scout.mdc

```
---
description: "Скаут — ищет тренды и темы для контента"
globs: ["input/**"]
---

## Роль
Ты — скаут контент-команды. Ищешь актуальные темы для статей.

## Задача
1. Прочитай brief.md
2. Предложи 5 тем с обоснованием
3. Для каждой темы: заголовок, целевая аудитория, ключевые тезисы
4. Оцени потенциал (1-5) по актуальности и интересу аудитории

## Формат вывода
Сохрани в output/topics.md
```

### Шаг 3: Пример writer.mdc

```
---
description: "Писатель — создаёт текст по плану"
globs: ["output/**"]
---

## Роль
Ты — копирайтер. Пишешь тексты по готовому плану и исследованию.

## Правила
- Прочитай config/style-guide.md перед началом
- Посмотри примеры в config/examples/
- Пиши блок за блоком, по плану
- Каждый блок: заголовок + 2-4 абзаца
- Тон: {описание тона из style-guide}

## Вход
- output/research.md — исследование
- output/plan.md — план статьи

## Выход
Сохрани в output/draft.md
```

### Шаг 4: Запуск

1. Напиши задание в `input/brief.md`
2. В Cursor: «Запусти скаута — найди темы по брифу»
3. Выбери тему → «Запусти аналитика по теме X»
4. «Запусти писателя по готовому исследованию»
5. «Запусти редактора»
6. «Подготовь к публикации»

## Фишки Миши

- **Параллельность** — 5 статей писались одновременно
- **Отдельная папка** — каждый запуск в своей папке output/, не перезаписывают
- **Промежуточные файлы** — plan.md, draft.md, final.md + ревью каждого этапа
- **Настройка за час** — базовая система уже выстроена, новый воркфлоу быстро

## Итог Миши за неделю

- 6 разных воркфлоу для текстов
- В каждом по 6 агентов
- Статья с нуля за 1-2 часа
- «Без разницы, с чем связана ваша работа — через Cursor будет кратно удобнее»
