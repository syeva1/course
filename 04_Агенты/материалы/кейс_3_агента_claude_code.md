# Кейс: 3 агента в Claude Code заменили команду разработки

> Источник: [Reddit r/micro_saas](https://www.reddit.com/r/micro_saas/comments/1rju8sd/i_replaced_my_dev_team_with_3_claude_code_agents/) | Канал: @misha_davai_po_novoi

## Суть

Один разработчик создал 3 агента в Claude Code, каждый со своей ролью и жёсткими границами. За 4 недели собрал работающий SaaS-сервис.

## Три агента

**Агент 1 — Бэкенд:**
- База данных
- Логика приложения
- Платёжные интеграции (Stripe, PayPal)

**Агент 2 — Фронтенд:**
- Страницы и интерфейс
- Кнопки, визуал
- Пользовательский опыт

**Агент 3 — Стратег:**
- Координация задач
- Маркетинг
- Аналитика

## Как общались между собой

Через **общую папку с markdown-файлами**:
- Задачи для каждой роли
- Логи решений
- Файлы с ошибками (чтобы не повторять ошибки друг друга)

## Результат за 4 недели

- Работающий сервис с приёмом платежей
- Настроенная аналитика
- Рекламный кабинет

## Как повторить в Cursor

1. Создай 3 файла `.mdc` — по одному на каждого "агента" (роль + границы + правила)
2. Заведи папку `_coordination/` с файлами:
   - `tasks_backend.md` — задачи для бэкенда
   - `tasks_frontend.md` — задачи для фронтенда
   - `tasks_strategy.md` — задачи для стратега
   - `errors_log.md` — общий лог ошибок
   - `decisions.md` — принятые решения
3. В каждом `.mdc` укажи: "Перед работой прочитай свои задачи из `_coordination/`. После работы обнови лог."
4. Запускай агентов по очереди или параллельно (через Worktree)

## Главный вывод

Мульти-агентная работа — это не магия. Это **чёткие роли + общая папка для координации**. Работает и в Claude Code, и в Cursor.
