# Структура проекта для AI-агентов

> Как организовать проект, чтобы Cursor/Claude Code работал максимально эффективно.
> Источник: @misha_davai_po_novoi

---

## Идеальная структура

```
project/
├── README.md              # Cursor читает ПЕРВЫМ — описание проекта
├── .cursor/
│   └── rules/             # .mdc правила (= агенты)
│       ├── general.mdc    # Общие правила для всего проекта
│       ├── code.mdc       # Правила для кода
│       ├── content.mdc    # Правила для текстов
│       └── review.mdc     # Правила для ревью
├── docs/
│   ├── architecture.md    # Как устроен проект
│   └── decisions/         # Почему приняли такое решение
│       └── 001-выбор-стека.md
├── tasks/
│   ├── todo.md            # Текущие задачи
│   └── lessons.md         # Уроки из ошибок (AI читает и не повторяет)
├── config/
│   ├── style-guide.md     # Стиль текстов / кода
│   └── examples/          # Примеры хорошего результата
├── src/                   # Код
├── tests/                 # Тесты
└── .env                   # Секреты (в .gitignore!)
```

## Зачем каждый файл

### README.md
Cursor читает первым. Чем подробнее — тем лучше код. Минимум:
- Что за проект
- Какие технологии
- Как запустить
- Структура папок

### .cursor/rules/*.mdc
Правила для AI. Cursor подтягивает автоматически по globs.
- `general.mdc` — стиль, язык, формат
- Специализированные — по типу задачи

### tasks/lessons.md
**Самый недооценённый файл.** Каждый раз, когда AI ошибся — запиши:
```markdown
## Урок 3: Не использовать inline styles
Дата: 2026-03-10
Ошибка: AI добавил inline CSS вместо классов
Правило: Всегда используй CSS-классы, никогда inline styles
```
AI прочитает и не повторит ошибку.

### config/examples/
Покажи AI, как выглядит хороший результат. Один пример стоит 100 слов инструкций.

### docs/decisions/
Записывай, ПОЧЕМУ выбрал конкретное решение. Когда AI предложит изменить — он увидит причину.

## Аналогия с Cursor

| Концепция | В Cursor |
|-----------|----------|
| CLAUDE.md | `.cursor/rules/general.mdc` |
| Skills | `.cursor/rules/*.mdc` с узкой специализацией |
| Вложенные контексты | .mdc с `globs: ["src/api/**"]` |
| lessons.md | `tasks/lessons.md` (Cursor читает автоматически) |

## Правила от Миши

1. **README.md — визитка.** Cursor читает первым. Не ленись.
2. **Один .mdc = одна роль.** Не мешай всё в один файл.
3. **Примеры > инструкции.** Покажи результат, а не описывай.
4. **lessons.md обновляй СРАЗУ.** После каждой ошибки — новый урок.
5. **Маленькие файлы > большие.** Cursor лучше работает с контекстом в 500 строк, чем в 5000.
